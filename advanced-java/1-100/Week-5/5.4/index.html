user-service​

Here, we used user-service to send message to kafka server

code​

  • Spring Boot app that sends message to the Apache Kafka
  • This Controller:
    • Receives HTTP requests (like from Postman or a browser).
    • Sends the given message to a Kafka topic called user-random-topic.
    • @RestController - This class handles HTTP request like GET/POST.
package com.codecrush.learnKafka.user_service.controller;

import lombok.RequiredArgsConstructor;
import lombok.extern.slf4j.Slf4j;
import org.springframework.http.ResponseEntity;
import org.springframework.kafka.core.KafkaTemplate;
import org.springframework.web.bind.annotation.*;

@RestController
@Slf4j
@RequestMapping("/users")
@RequiredArgsConstructor
public class UserController {

    private final KafkaTemplate<String, String> kafkaTemplate;

    @PostMapping("/{message}")
    public ResponseEntity<String> sendMessage(@PathVariable String message){
        kafkaTemplate.send("user-random-topic", message);
        return ResponseEntity.ok("Message queued");
    }
    @GetMapping("/hello")
    public ResponseEntity<String> testHello() {
        return ResponseEntity.ok("Hello endpoint working!");
    }

}

What is happening above?​

  • When you open Postman (or your frontend) and send a POST request to: http://localhost:8080/users/HelloWorld
  • It takes HelloWorld as a message.
  • It sends that message to a Kafka topic called user-random-topic.
  • Then it replies: "Message queued"
  • run kafbat ui to look topic created and message and delete it.
  • Here, when we are hitting endpoints then only topic is being created.

public class KafkaTopicConfig {

    @Value("${kafka.topic.user-random-topic}")
    private String KAFKA_RANDOM_USER_TOPIC;

Why do all messages go to the first partition?​

  • Kafka's partitioning logic depends on the key of the message:
    • If you send a message without a key, Kafka uses a round-robin or sticks to partition 0 depending on the producer's implementation and version.
    • If you send a message with a key, Kafka uses the key’s hash to decide the partition (so messages with the same key always go to the same partition).
  • we can also do
    • kafkaTemplate.send(topicName, partitionNumber, key, message);
    • we can give key, message
    • only message
    • partition, key, message
    • if partition are, then giving the key must be needed if not provided write null.

package com.codecrush.learnKafka.user_service.controller;

import lombok.RequiredArgsConstructor;
import lombok.extern.slf4j.Slf4j;
import org.springframework.beans.factory.annotation.Value;
import org.springframework.http.ResponseEntity;
import org.springframework.kafka.core.KafkaTemplate;
import org.springframework.web.bind.annotation.*;

import java.util.UUID;

@RestController
@Slf4j
@RequestMapping("/users")
@RequiredArgsConstructor
public class UserController {

    @Value("${kafka.topic.user-random-topic}")
    private String KAFKA_RANDOM_USER_TOPIC;

    @Value("${kafka.partition.count}")
    private int TOTAL_PARTITIONS;

    private final KafkaTemplate<String, String> kafkaTemplate;

    @PostMapping("/{message}")
    public ResponseEntity<String> sendMessage(@PathVariable String message){
        // Generate random key for each message
//        String key = UUID.randomUUID().toString();
////        kafkaTemplate.send(KAFKA_RANDOM_USER_TOPIC,2,message, message);
//        kafkaTemplate.send(KAFKA_RANDOM_USER_TOPIC,2,key, message);
//        return ResponseEntity.ok("Message queued");


        String key = UUID.randomUUID().toString();
        int hash = key.hashCode();
        int partition = Math.abs(hash % TOTAL_PARTITIONS);

        log.info("Generated Key: {}", key);
        log.info("HashCode: {}", hash);
        log.info("Total Partitions: {}", TOTAL_PARTITIONS);
        log.info("Target Partition (via hash): {}", partition);

        kafkaTemplate.send(KAFKA_RANDOM_USER_TOPIC, partition, key, message);
        return ResponseEntity.ok("Message queued with key: " + key);
    }
    @GetMapping("/hello")
    public ResponseEntity<String> testHello() {
        return ResponseEntity.ok("Hello endpoint working!");
    }

}

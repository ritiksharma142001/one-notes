Setting up a Spring Boot Project

Setup Project​

  • Intellij IDE(Integrated Development Environment) - Community Edition
  • Spring Initializr

Download Spring Boot Project Using Spring Initializr​

  • Spring Initializr
  • Configure
OptionValue/Explanation
ProjectMaven (used widely in industry)
LanguageJava
Spring BootChoose latest stable version (e.g., 3.3.0) SNAPSHOT version are not stable
GroupUnique org/group name (e.g., com.codecrush.ritik)
ArtifactName of your app (e.g., introduction-to-springboot)
Name(Optional) Auto-filled from Artifact

Package NameAuto-generated from Group + Artifact

Add Dependencies​

Add required Spring Boot starter dependencies:

  • Spring Web → for building REST APIs
  • Spring Data JPA → for database access
  • Spring Security → for securing your app
  • Thymeleaf → if you want to use HTML templates

These are "starter" dependencies, meaning you don’t need to configure everything manually.

Download and Open Project​

  1. Click Generate – it will download a .zip file.
  2. Extract the .zip file.
  3. Open IntelliJ IDEA.
  4. Click Open → navigate to the extracted folder.
  5. Click Trust Project when prompted.

Spring Boot Project Structure & Configuration Overview​

Project Structure

  • Your Java project is auto-generated using Spring Initializr.
  • It contains default packages and files essential for a Spring Boot application.

src/main/resources Used to store:

  • Static files (.html, images, JS/CSS under static/)
  • Templates (Thymeleaf, Freemarker, etc.)
  • Configuration files (application.properties or application.yml)

Changing Server Port By default, Spring Boot runs on port 8080. To change:

# application.properties
server.port=9000

This tells the embedded Tomcat server to run on port 9000.

Spring Boot Configurations You can customize many properties via application.properties.

debug=true

This enables more detailed logging when running the application.

  • To explore more:
    • Go to Spring Boot documentation
    • Search for: spring boot common application properties
    • Example site: https://docs.spring.io/spring-boot/docs/current/reference/html/application-properties.html

** Test Folder Structure**

  • src/test/java: For unit and integration test cases
  • Best practice: Test files should mirror the structure of the main source files.
com.example.demo.MyService.java
com.example.demo.MyServiceTest.java

Test-Driven Development (TDD):

  • Write tests before or alongside your code.
  • Ensures each piece of functionality is tested.
  • Widely followed in big tech companies.

target/ Folder

  • Auto-generated by Maven.
  • Stores compiled files and the final .jar or .war file.
  • No need to edit files manually here.

Other Files

  • .gitignore: Lists files/folders to be ignored by Git.
  • HELP.md: Basic guide or help documentation.
  • mvnw, mvnw.cmd: Maven Wrapper scripts — allow using Maven without installing it locally.

pom.xml – Project Object Model Similar to package.json in Node.js projects.

  • Used to:
    • Define project metadata (name, version, description, etc.)
    • Declare dependencies
    • Define build plugins, Java version, and more
<dependency>
  <groupId>org.springframework.boot</groupId>
  <artifactId>spring-boot-starter-web</artifactId>
</dependency>
  • This gets downloaded from Maven Central Repository.

External Libraries

  • When you add dependencies via pom.xml, Maven downloads required jars automatically.
  • These appear in the IDE (like IntelliJ or Eclipse) under External Libraries.
  • Example: Spring Web brings dependencies like:
    • spring-web
    • jackson
    • tomcat

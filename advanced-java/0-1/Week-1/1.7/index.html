Maven

Introduction to Maven in Spring Boot​

Introduction to Maven​

  • Maven is a build automation and project management tool.
  • Though often associated with Spring, Maven is not limited to Spring Boot or Spring Framework.
  • It is widely used to build and manage Java applications.

Role of Maven in Java Projects​

  • Maven is a core tool in Java development.
  • It:
    • Helps build Java applications.
    • Manages dependencies.
    • Facilitates project packaging and deployment.
    • Can work with any Java-based framework, including Spring Boot.

Why Use Maven?​

  • Maven helps:
    • ompile the source code.
    • Run tests.
    • Package applications (e.g., into .jar or .war files).
    • Deploy artifacts to remote repositories (e.g., Maven Central).
    • Manage third-party dependencies by pulling them from repositories.

Maven in Spring Boot​

  • Spring Boot leverages Maven to:
    • Resolve and download required dependencies.
    • Build and run the application.
    • Generate an executable .jar file.
    • Package everything in a single deployable artifact.

Maven Central Repository​

  • Maven Central is a remote repository that hosts:
    • Thousands of reusable Java libraries.
    • Spring Boot dependencies.
  • Maven automatically downloads required libraries from there based on the configuration in pom.xml.

Maven Analogy (Chef Example)​

To better understand Maven, consider the following analogy:

ComponentRole in AnalogyRole in Spring Boot
Maven (Chef)Prepares food using recipe and ingredientsBuilds the app using configuration + dependencies
pom.xml (Recipe)Tells the Chef what to make and what is neededConfiguration file containing dependencies
DependenciesIngredients needed to cook foodExternal libraries pulled into your project
Built ApplicationFinal cooked dishFinal executable .jar or .war

Project & Dependency Management​

Lifecycle and Role of Maven​

  • Maven has a lifecycle that automates various phases of application development:
    • Compile
    • Test
    • Package
    • Install
    • Deploy
  • Maven is not just a build automation tool — it also acts as a:
    • Project management tool.
    • Dependency management tool.

Standardized Project Management​

  • Maven provides a standard structure for Java projects.
  • It manages:
    • Project structure.
    • Dependencies.
    • Build process and plugins.
  • All of this is handled via a declarative, XML-based approach.

Introduction to pom.xml​

  • Maven uses a special XML configuration file called pom.xml:
    • POM stands for Project Object Model.
  • This file defines:
    • Java version.
    • Project metadata (name, version, packaging, etc.).
    • Dependencies.
    • Plugins.
    • Repositories.

Dependency Management Example​

  • Spring Boot and Spring Framework modules (like spring-core, spring-mvc, etc.) are added as dependencies in pom.xml.
  • Developers just declare them — Maven handles:
    • Downloading dependencies from Maven Central Repository.
    • Adding them to the project.
    • Resolving any required transitive dependencies.

Maven Central Repository​

  • Spring Boot starter dependencies (like spring-boot-starter-web) are part of the Maven ecosystem.
  • These are hosted on the Maven Central Repository, from where:
    • Maven downloads and adds them to your project automatically.

Developer Experience​

  • The developer only needs to:
    • Configure pom.xml.
    • Specify project metadata and dependencies.
  • Maven handles the rest:
    • Downloading dependencies.
    • Managing project lifecycle.
    • Building the application.

Build Automation​

Understanding Build Automation​

  • Maven not only manages dependencies but also automates the entire build process.
  • You don’t need to manually manage dependencies or compile multiple files separately.
  • Maven uses a build lifecycle to perform various development tasks automatically.

Maven Build Lifecycle​

  • Maven follows a standard build lifecycle, which consists of various phases, also referred to as goals:
    • validate – Checks if the project structure and configuration are valid.
    • compile – Compiles the source code of the project.
    • test – Runs the unit tests.
    • package – Packages the compiled code into a deployable format like .jar or .war.
    • install – Installs the packaged project to the local Maven repository.
    • deploy – Copies the final package to a remote repository for sharing.

Automation Process​

  • Maven automates:
    • Compilation of all Java classes.
    • Testing of all test classes.
    • Packaging into deployable artifacts like .jar or .war files.
  • Maven handles inter-class dependencies and builds everything into a single executable file.

Portable JAR File Creation​

  • After the build process, Maven generates a .jar file:
    • This file can be shared or deployed.
    • Anyone with Java installed can run the JAR file using the command:
java -jar your-application.jar

Why Use Maven Instead of Manual Compilation​

  • Without Maven, you would need to:
    • Compile each Java class manually.
    • Ensure correct order and dependencies manually.
  • Maven:
    • Compiles all files in one go.
    • Manages dependencies automatically.
    • Reduces developer effort significantly.

Running Test Cases​

  • Use the Maven goal:
mvn test
  • It runs all the test classes in the project.
  • Ensures code quality and verifies functionality automatically.

Spring Boot and Maven Integration​

  • Spring Boot applications typically include:
    • Starter dependencies (e.g., spring-boot-starter-web) in the pom.xml.
    • Maven uses these to fetch all related libraries from Maven Central.
  • Maven integrates seamlessly with Spring Boot to:
    • Handle dependencies.
    • Build and package the application.
    • Automate testing and deployment.

Running Maven Commands using mvnw (Maven Wrapper)​

Using the Maven Wrapper (mvnw)​

  • Spring Boot projects often come with a Maven Wrapper script (mvnw or mvnw.cmd) included.
  • This wrapper allows you to run Maven commands without installing Maven globally on your system.

What is mvnw?​

  • mvnw (for Unix/Linux/macOS) or mvnw.cmd (for Windows) is a shell script that:
    • Downloads the correct version of Maven.
    • Runs the specified Maven command.
  • Useful when sharing projects with others — they don’t need to install Maven separately.

Running Maven Commands via Terminal​

  • You can open the terminal in your project root directory and run Maven commands using:
./mvnw <command>
  • For example:
./mvnw compile

What does ./mvnw compile do?​

  • This command compiles all the source files in the project.
  • Maven processes your .java files and generates .class files.
  • These compiled files are stored in the target/classes directory.

Verifying Compilation​

  • After running ./mvnw compile, navigate to:
target/classes/
  • You will see all the compiled .class files corresponding to your Java classes.
  • This confirms that the Java code was compiled successfully.

Why Use mvnw compile?​

  • No need to manually compile each Java file.
  • Ensures all dependencies and configuration settings in pom.xml are respected.
  • Great for consistency across teams or CI/CD pipelines.

Purpose of ./mvnw package​

  • The package command is used to:
    • Compile the source code.
    • Bundle the compiled classes and resources into a JAR (or WAR) file based on your project configuration.

How to Run the Command​

./mvnw package
  • This command uses the Maven Wrapper (mvnw) to execute the package phase.
  • It compiles the code and then creates a .jar file (because the packaging type is set to jar in pom.xml).

Where is the JAR File Created?​

  • After running the command, navigate to: target/
  • Inside the target folder, you’ll find the generated JAR file. Example: introduction-to-spring-boot-0.0.2.jar

Versioning the JAR File​

  • The version of the JAR file (e.g., 0.0.2) is defined in the <version> tag of the pom.xml file.
  • You can change the version to reflect different types of changes:
    • 1.0.0 → Major release.
    • 1.0.1 → Minor update.
    • 1.0.1-patch1 → Patch-level fixes.

Distributing the JAR File​

  • The generated .jar file can be shared via:
    • Email
    • WhatsApp
    • Any file-sharing method
  • Your friends or teammates don’t need the source code — just the JAR file.

Running the JAR File​

  • Anyone with Java installed can run the JAR using the following command:
java -jar target/introduction-to-spring-boot-0.0.2.jar
  • This will start the Spring Boot application locally.

Benefits

  • No need to share code.
  • No Maven installation required. -Portable application: Any Java-enabled system can run the JAR file.

Maven clean Command​

  • The clean command is used to remove all files generated by the previous build, especially the target/ folder.
  • This helps ensure that your next build is fresh and doesn’t include outdated or cached compiled files.
./mvnw clean

What Happens:

  • Deletes the target/ directory.
  • Frees up space and removes previously compiled .class and .jar files.
  • Helpful before running package or install commands to avoid unexpected behavior.

Maven test Command​

  • The test command runs all unit tests written under the src/test/java directory using testing frameworks like JUnit or Mockito.
  • Helps ensure code correctness and reliability.
./mvnw test

Test Case Location:

  • Test cases are typically defined inside:
src/test/java
  • In this session, sample test methods are placed within these files.

Debug Logs and Output​

  • Initially, the application.properties file had:
debug=true

which led to verbose logs.

  • To reduce log clutter, the debug property was commented out.

Running Test Cases Again​

  • After commenting out the debug line, test cases were re-executed using:
./mvnw test
  • As a result:
    • The test case executed successfully.
    • Any System.out.println() (e.g., Anu is running test) statements inside the test method were printed to the console.
    • Maven displayed the test summary:
Tests run: 1, Failures: 0, Errors: 0, Skipped: 0
@Test
void contextLoads() {
    System.out.println("My name");
}

mvn install Command​

  • Used to install the built .jar or .war file into the local Maven repository (~/.m2/repository).
  • Useful when:
    • You have multiple modules in a project.
    • You want to use a jar built from one module as a dependency in another module.
./mvnw install

mvn deploy Command​

  • Used to deploy your project artifacts to a remote Maven repository, such as Nexus or Artifactory.
  • You need to configure:
    • repository URL
    • username and password
  • Repository credentials are typically added in settings.xml.
./mvnw deploy

mvn spring-boot:run Command​

  • Used to run the Spring Boot application directly from the source code without building the jar file manually.
  • The IDE run button internally uses this command.
./mvnw spring-boot:run
  • Starts the embedded server (e.g., Tomcat) and runs the app (e.g., on port 9000).

mvn spring-boot:build-image Command​

  • Used to build a Docker image of your Spring Boot application using Cloud Native Buildpacks.
  • You do not need to write a Dockerfile.
./mvnw spring-boot:build-image
  • Docker must be installed and running for this to work.

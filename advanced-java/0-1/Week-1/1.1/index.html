Introduction to Spring Framework and IOC Container

  • Notes

What is Spring Framework?​

  • Spring Framework is a Java-based framework that helps developers build applications more easily. It was first introduced in 2003 by Rod Johnson.

Why was Spring Framework needed?​

  • Before Spring, developers used Java EE (Java Enterprise Edition) to build things like:
    • Web applications
    • Authentication systems
    • Batch processing
  • But Java EE had a big problem:
  • Developers had to do a lot of configuration before they could even start writing their actual business logic.

Real-Life Example: Buying a House​

  • Imagine you buy a new house:
    • With Java EE, you get an empty house (just walls and rooms).
    • You must furnish it, add electricity, lights, furniture, etc., before you can live in it.
  • This "furnishing" is like configuring servers, databases, logging, authentication, and more in Java EE. It could take 2–3 months just to set things up!

How Spring Framework Helped?​

  • Spring didn’t give you a fully furnished house right away, but it gave you a butler/helper (a tool) who helps you:
    • Set up things faster
    • Add libraries
    • Manage your database and server
    • Inject dependencies automatically
  • This helper in Spring is called the IoC Container.

Key Features of Spring Framework​

  1. IoC Container (Inversion of Control)

    • It manages objects (called beans) for you.
    • Helps in Dependency Injection (automatically giving objects what they need to work).

  2. Dependency Injection

    • Helps create loosely coupled code.
    • You don’t need to manually create objects everywhere.

  3. AOP (Aspect-Oriented Programming)

    • Helps add common things like logging, security, and transactions in one place.

  4. Bean Management

    • Automatically creates and manages objects (beans) in your application.

  5. XML-based configuration (in early versions)

    • You could tell Spring what to do using XML files.

Evolution from Spring Framework to Spring Boot​

Challenges with Spring Framework

  • Even with the introduction of the IoC container (Inversion of Control) in Spring Framework (2003), developers still spent a significant amount of time configuring applications.
  • Tasks that previously took 2–3 months could now be done in 7–10 days, but this was still not ideal.
  • Manual configuration remained a major bottleneck.

Enter: Spring Boot (2014)​

  • Spring Boot was introduced in 2014 to simplify and speed up Spring-based application development.
  • It is not a separate framework—rather, it's built on top of Spring Framework.
  • Spring Boot provides a production-ready, pre-configured environment for developing Spring applications quickly.

The Fully Furnished House Analogy​

With Spring Boot, think of getting a fully furnished house, where:

  • The application is pre-configured with sensible defaults, so you can start coding immediately.
  • You get:
    • A built-in, production-ready server setup
    • Default database configurations
    • Pre-configured logging, monitoring, and more

Key Features of Spring Boot​

  • Starter Dependencies: Predefined dependency sets that simplify build configuration.
  • Auto-Configuration: Automatically configures Spring components based on the dependencies and environment.
  • IoC Container: Just like in Spring Framework, Spring Boot uses the powerful IoC container (your "butler").
  • Minimal Setup Required: Developers can start writing business logic from Day 1 without worrying about boilerplate configuration.

Developer Productivity Boost​

  • Developers no longer have to spend days configuring the project setup.
  • Simply generate a project using Spring Initializr, and you're ready to build.
  • This is the true power of Spring Boot—rapid development with minimal setup.

Timeline & History​

  • 2003: Rod Johnson created the Spring Framework.
  • 2009: Spring was acquired by VMware.
  • 2014: VMware released Spring Boot, built on top of Spring Framework.
  • Spring Boot revolutionized Spring application development by removing the need for manual configuration.

Spring Boot provides​

  • Auto-configured components
  • Production-ready setups
  • Rapid development workflows
  • It eliminates the overhead of configuration and lets developers focus on what matters most: business logic.

Introduction to Spring Framework​

Before diving deep into Spring Boot, it’s important to understand the Spring Framework itself. What is the Spring Framework?

  • Spring is a Dependency Injection (DI) framework for building loosely coupled Java applications.
  • A loosely coupled system allows components to interact with each other with minimal dependencies, making the system easier to maintain and extend.
  • We'll explore the concept of loose coupling in more detail later, but for now, just know that Spring helps achieve this through Dependency Injection.

Purpose of Spring Framework​

  • Spring simplifies the development of Java EE (Enterprise Edition) applications.
  • Before Spring, developers relied heavily on Java EE (formerly J2EE) to build web services and enterprise solutions, which required a lot of manual configuration.
  • Spring made this process much easier by:
    • Reducing boilerplate code
    • Providing a powerful IoC container
    • Supporting configuration through XML files (in earlier versions)

Role of the IoC Container​

  • The IoC (Inversion of Control) container in Spring handles the management of objects and dependencies.
  • This means a lot of the heavy lifting is done by the framework, allowing developers to focus on business logic.

Build Applications with POJOs​

  • Spring enables developers to create applications using Plain Old Java Objects (POJOs).
  • Even complex enterprise applications can be built entirely using POJOs.
  • You can add enterprise-level features to these POJOs non-invasively, meaning:
    • Your core logic remains clean and free from framework-specific code.
    • Features like logging, security, metrics, health checks, and actuators can be added without modifying your business logic.

Enterprise Services Made Easy​

  • With Spring, you can add:
    • Health checks
    • Security mechanisms
    • Logging
    • Monitoring and metrics
  • All of these can be integrated seamlessly without tightly coupling them to your application logic.

A Brief History​

  • Spring Framework was created by Rod Johnson in 2003.
  • It was later acquired by VMware, and VMware continues to maintain it.
  • Today, Spring has:
    • A large and active developer community
    • Extensive official documentation
    • Robust support through Spring Boot

Important Components of the Spring Framework​

The Spring Framework is composed of several mini frameworks or modules, each designed to solve specific problems in Java application development.

Modular Design of Spring​

Spring is not a single monolithic framework. Instead, it provides a wide range of modules such as:

  • Core Container: Contains essential components like Beans, Dependency Injection, and the ApplicationContext.
  • Testing Framework: Helps in writing unit and integration tests for Spring applications.
  • Expression Language (SpEL): A powerful language for querying and manipulating object graphs at runtime. Note: You don’t need to study each module in depth if you're using Spring Boot.

Why You Don’t Need to Learn Every Module​

  • With the rise of Spring Boot, much of the low-level configuration and boilerplate code from the Spring Framework is now abstracted away.
  • Spring Boot provides a higher-level abstraction on top of the Spring Framework, so developers can focus on writing business logic, not infrastructure code.

Examples of Abstractions in Spring Boot​

  • You no longer need to manually configure JDBC. Spring Boot handles it for you.
  • You don’t need to worry about setting up ORM (Object-Relational Mapping) either. Spring Boot integrates Hibernate and adds Spring Data JPA on top.
  • Configuration for servlets, data sources, security, and many other components is already done by default in Spring Boot.

Built by the Experts​

All the heavy lifting and base-layer code has already been built by expert Spring developers. You just use the APIs and configurations they’ve provided, so you can hit the ground running with your own application logic.

Key Components We Will Cover in This Course​

We will cover the essential modules required to build a real-world Spring Boot application:

  1. Core Container
    • Handles Dependency Injection and Beans Management
    • Covered in the upcoming module
  2. AOP (Aspect-Oriented Programming)
    • Allows modularization of cross-cutting concerns like logging, security, and transaction management
  3. JDBC Module
    • Enables database interactions (automatically configured in Spring Boot)
  4. Web Module
    • Used to build REST APIs and web applications
  5. Testing Module
    • Provides support for writing unit and integration tests

summary

  • The Spring Framework offers multiple modules, but you don't need to configure everything manually anymore.
  • Spring Boot offers an abstraction layer that handles most of the setup for you.
  • In this course, we’ll focus only on the core components needed for real-world Spring Boot development.

Spring Framework Introduction – Core Components & IoC Container​

1. What is Spring Framework? The Spring Framework is a comprehensive, modular framework for Java-based enterprise applications. It contains many sub-frameworks (called “projects” or “modules”), such as:

  • Spring Core: Dependency Injection & Bean Container
  • Spring AOP: Aspect-Oriented Programming
  • Spring JDBC: Simplified database access
  • Spring Web: For building web applications
  • Spring Test: Testing support Today, we mostly use Spring Boot, which simplifies the usage of the Spring Framework by removing a lot of manual configuration and boilerplate code.

2. Why Spring Boot is Preferred?

  • Spring Boot provides an abstraction over the Spring Framework.
  • You don’t need to configure low-level details like JDBC or ORM manually.
  • Features like Hibernate, Spring Data JPA, and auto-configuration are already integrated.
  • Focus stays on business logic rather than configuration.

3. What is IoC Container? IoC stands for Inversion of Control. Definition: The IoC Container is the core component of the Spring Framework. It manages:

  • The creation of objects (called beans)
  • Wiring dependencies
  • Managing object lifecycle

How does it work?

  • You write plain Java classes (POJOs).
  • You add configuration using:
    • XML files
    • Java annotations
    • application.properties or application.yml
  • Spring uses this configuration to:
    • Create objects
    • Inject dependencies
    • Control when and how objects are used

4. Features Inside IoC Container

FeatureDescription
Dependency Injection (DI)Automatically provides class dependencies (e.g., service to controller).
Event-Driven ProgrammingExecutes code based on events (e.g., button click handlers).
AOP (Aspect-Oriented Programming)Helps in cross-cutting concerns like logging, security, etc.
Bean Lifecycle ManagementSpring manages object creation to destruction lifecycle.

5. Benefits of Using IoC

  • Loose coupling: Code is flexible and testable.
  • Environment separation: Change config, not code (e.g., dev vs prod).
  • Maintainability: Easier to manage large applications.

6. Official Definition: “In the Spring Framework, the IoC container is responsible for managing the components of an application and injecting dependencies into them. The container creates the objects (beans), wires them together, configures them, and manages their complete lifecycle.”

Summary:

  • Spring Boot builds on Spring Framework and makes it easy to develop Java apps.
  • The IoC container is the heart of Spring—responsible for Dependency Injection, bean creation, lifecycle management, etc.
  • You don’t have to write a lot of configuration—Spring Boot does that for you.

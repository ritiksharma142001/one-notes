Lamdas & Streams

Introduction to Lambda Expressions​

What is a Lambda Expression?​

  • Lambda is a short and concise way to represent a method using an anonymous function.
  • Lambdas are typically used to implement functional interfaces.
  • A functional interface is an interface that contains only one abstract method.
  • Functional interfaces were introduced in Java 8 to support Lambda expressions.

Creating and Using a Functional Interface​

Example: Functional Interface​

  • Let's define a functional interface:
interface Walkable {
    int walk(int steps);
}

Implementing the Interface​

  • You can implement this interface using a class:
class WalkFast implements Walkable {
    @Override
    public int walk(int steps) {
        System.out.println("Walking fast " + steps + " steps");
        return 2 * steps;
    }
}

Creating an Instance of the Interface​

  • Using the class implementation:
Walkable obj = new WalkFast();
int result = obj.walk(4); // Output: Walking fast 4 steps

Using an Anonymous Class (Tedious Way)​

You can also use an anonymous class to implement the interface:

Walkable obj = new Walkable() {
    @Override
    public int walk(int steps) {
        return steps * 2;
    }
};
  • However, this approach is lengthy and less readable.

Why Lambdas?​

  • Problem with Traditional Approach
    • Creating multiple classes or anonymous classes for small functionalities is verbose.
    • In real-world applications, often you only need a one-time use of an interface implementation.
  • Solution: Use Lambda Expressions
    • Lambdas simplify the creation of interface implementations.
    • We'll explore how to rewrite the above examples using Lambda expressions.

code​

public class Main {
    public static void main(String[] args) {
        Walkable obj = new WalkFast();
        obj.walk(4);
    }
}

interface Walkable{
    int walk(int steps);
}

class WalkFast implements Walkable {

    @Override
    public int walk(int steps) {
        System.out.println("Walking fast "+steps+" steps.");
        return 2*steps;
    }
}

Understanding Functional Interfaces and Lambda Expressions in Java 8​

  • Java developers identified a common use case:
    • When interfaces have only one method, it is often unnecessary to create separate class implementations or anonymous classes.
  • Older Way of Implementing Interfaces:
    • Define a class implementing the interface.
    • Override the method(s) of the interface.
    • Create an object using new keyword.
    • This is useful when interfaces have multiple methods.
  • Newer Way using Lambda Expressions:
    • If an interface has only one abstract method, it's called a functional interface.
    • You can annotate it with @FunctionalInterface (optional but good practice).
    • This enables a more concise syntax using Lambda expressions.

How Lambda Works:​

  • Instead of:
Walkable object = new Walkable() {
    @Override
    public int walk(int steps) {
        return 2 * steps;
    }
};
  • You can write:
Walkable object = (int steps) -> {
    return 2 * steps;
};

Explanation:​

  • No need to use new keyword.
  • No need to mention method name (walk), as compiler already knows.
  • You only define parameter(s) and the method body.

What is a Stream in Java?​

  • A Stream is a sequence of elements (like a list) that supports various operations to perform computation in a declarative way (what to do, not how to do it).
  • Think of a stream like flowing water — once it flows past a point, it cannot go back. Similarly, in Java, once a stream is consumed (used), it cannot be reused.

Why Streams?​

  • Helps process collections (like List, Set) in a functional style
  • Supports method chaining (like .filter().map().sorted())
  • Helps write clean and concise code
  • Great for memory management, since streams can be closed and reused efficiently

Sorting & Pagination

Key Concepts Covered in the Session​

Data Retrieval and Filtering:​

  • Discussed how to retrieve data using Spring Data JPA.
  • Explained filtering data based on multiple fields (e.g., title, createdAt).
  • Demonstrated using various keywords for filtering:
    • after, before, greaterThan, lessThan, like, containing, ignoreCase, equals, etc.

Sorting Data:​

  • Introduced how to sort the retrieved data.
  • For example, on an e-commerce platform like Amazon, sorting can be done based on:
    • Featured products
    • Price: Low to High or High to Low
  • Explained how to implement sorting based on multiple entity fields using Spring Data JPA.

Pagination:​

  • Explained pagination with real-world examples like Amazon.
  • Instead of loading all items (e.g., all watches), only one page of data is fetched at a time.
  • Users can navigate to the next page to load more data.

Importance of Pagination:​

  • Prevents overloading the user with large amounts of data.
  • Enhances performance and reduces memory usage.
  • Provides a smooth user experience by loading data in chunks.

Pagination Strategies:​

  • Common front-end approaches:
    • Pagination bar (e.g., page 1, 2, 3)
    • Infinite scroll
  • Back-end handles pagination logic, usually involving:
    • Page number (page)
    • Page size (size)
    • Optional: Sort parameters

Backend Support in Spring Data JPA:​

  • Spring provides built-in support for pagination and sorting via:
    • Pageable
    • Page<T> interface
  • Developers don’t need to write complex logic manually.

Topics to Be Covered in the Session:​

  • Implementing sorting using Spring Data JPA.
  • Implementing pagination using Pageable and Page.
  • Returning paginated and sorted responses from the backend.

Sorting in Spring Data JPA​

Sorting in Multiple Ways​

  • Sorting can be achieved in multiple ways in Spring Data JPA.
  • One common method is using ORDER BY clause in your query methods.

Example Query Method​

  • Suppose you want to find products by title and sort them by price:
List<Product> findByTitleOrderByPrice(String title);
  • This will return all products with the given title, sorted by price in ascending order.

Controller Setup for Testing Sorting​

  • A ProductController was created to test sorting via API.
  • Annotation setup:
@RestController
@RequestMapping("/products")
public class ProductController { ... }

Repository Injection (Without Service Layer)​

  • Instead of implementing a full MVC architecture for simplicity, the repository was injected directly into the controller:
private final ProductRepository productRepository;

public ProductController(ProductRepository productRepository) {
    this.productRepository = productRepository;
}
  • While not production-ready, this avoids unnecessary boilerplate during learning.

Initial Sorting Example​

  • A GET endpoint method was created that:
    • Returns a list of products with a specific title (e.g., "CocaCola" or "Massa")
    • Sorts them by price:
@GetMapping
public List<Product> getAllProducts() {
    return productRepository.findByTitleOrderByPrice("Massa");
}
  • Output showed only one matching record, so sorting was not clearly visible.

Improved Example – Sorting All Products​

  • The query was changed to fetch all products, not filtered by title.
  • Updated method:
List<Product> findAllByOrderByPrice();
  • The corresponding controller method:
@GetMapping
public List<Product> getAllProducts() {
    return productRepository.findAllByOrderByPrice();
}
  • This method returns all products sorted by price.

Query Naming Notes​

  • If no filter field is applied, it is better to name the method:
findAllByOrderByPrice()
  • instead of findBy.
  • Using findBy with no conditions is acceptable for small projects, but findAllBy... improves readability.

Sorting Verification​

  • When the API endpoint was tested:
    • Products were returned sorted by price.
    • Earlier, default sorting was by ID.
    • After applying order by price, the products came in ascending price order: e.g., 10.5, 11, 12, ...

Dynamic Sorting in Spring Data JPA Using Sort Parameter​

Limitation of Static OrderBy Methods​

  • Using findBy...OrderByFieldName makes your method tightly coupled to a specific field.
  • If you need to sort by different fields (e.g., title, sku, quantity), you'll have to create separate methods for each.
  • This approach is not scalable and leads to code duplication.

Solution: Use the Sort Parameter​

  • Spring Data JPA allows dynamic sorting by passing a Sort object.
  • This eliminates the need to create separate methods for each sort field.

How to Use the Sort Class​

  • You can overload your repository method to accept a Sort parameter:
List<Product> findAll(Sort sort);

Update Controller to Accept Sort Parameter​

  • Modify your controller method to accept a query parameter from the client:
@GetMapping
public List<Product> getAllProducts(
    @RequestParam(defaultValue = "id") String sortBy
) {
    Sort sort = Sort.by(sortBy);
    return productRepository.findBy(sort);
}
  • Now, if the user hits /products?sortBy=title, results will be sorted by title.

Default Sort Fallback​

  • If no query parameter is passed, sorting will fall back to the default field: id

Changing Sort Direction (ASC/DESC)​

  • You can specify the sort direction:
Sort sort = Sort.by(Sort.Direction.DESC, sortBy);

You can also make both sortBy and direction dynamic:​

@RequestParam(defaultValue = "id") String sortBy,
@RequestParam(defaultValue = "ASC") String direction
  • And then:
Sort sort = Sort.by(Sort.Direction.fromString(direction), sortBy);

Multi-Level Sorting​

  • If two records have the same value for the first field, you can add fallback sorting fields:
Sort sort = Sort.by(Direction.DESC, sortBy, "price", "quantity");
  • This will:
    • Sort by sortBy (e.g., title)
    • Then by price
    • Then by quantity

Example Usages​

  • /products?sortBy=title → Sorted by title (ascending)
  • /products?sortBy=sku → Sorted by SKU
  • /products?sortBy=price → Sorted by price
  • /products?sortBy=quantity&direction=DESC → Sorted by quantity in descending order
  • /products?sortBy=title&direction=DESC → Sorted by title in descending order

Result Confirmation​

  • Sorting works correctly:
    • When sortBy=price&direction=DESC → Highest price appears first.
    • When sortBy=quantity → Products are sorted by quantity in ascending order.
    • You can sort dynamically on any field: title, sku, price, quantity, etc.

Advanced Sorting in Spring Data JPA using Sort.Order​

Another Way to Create Sort Object​

  • You can also use Sort.Order to define sorting criteria:
Sort sort = Sort.by(Sort.Order.desc("fieldName"));
  • This is an alternative to using Sort.by("fieldName").

Example: Sort by One Field​

  • Create a Sort object:
Sort sort = Sort.by(Sort.Order.desc(sortBy));
  • Use it in your repository method:
productRepository.findBy(sort);
  • This will sort the results in descending order based on the sortBy field.

Sorting by Multiple Fields​

  • You can chain multiple sorting criteria:
Sort sort = Sort.by(
    Sort.Order.desc(sortBy),
    Sort.Order.asc("price")
);
  • This means:
    • First, sort by sortBy field in descending order.
    • If multiple items have the same value, then sort by price in ascending order.

Real Use Case Example​

  • Suppose two products have the same quantity.
  • You can further sort them by title:
Sort sort = Sort.by(
    Sort.Order.desc("quantity"),
    Sort.Order.asc("title")
);
  • Example:
    • Quantity = 100 for both Coca-Cola and Twix.
    • Then sort them alphabetically by title.

Flexible & Use Case Driven​

  • You can add as many fields as you want for secondary or fallback sorting.
  • Sorting can be customized based on:
    • quantity, title, price, sku, etc.
    • Order can be either ASC or DESC.

Verify with Example Data​

  • Coca-Cola and Twix had the same quantity.
  • Sorted titles:
    • In ascending: Coca-Cola before Twix.
    • In descending: Twix before Coca-Cola.

Recommendation​

  • Prefer using Sort parameters over method names:
    • It's more flexible.
    • Supports dynamic field selection.
    • Allows multiple levels of sorting.
    • Reduces the need to create multiple repository methods.

Pagination in Spring Data JPA​

What is Pagination?​

  • Pagination refers to breaking large data sets into smaller chunks or "pages".
  • For example, if you have 1 million product records, you may want to retrieve just 25 at a time.
  • That subset is called a Page.

Key Interfaces and Classes in Spring Data JPA​

Page

  • Represents a single page of data.
  • Includes:
    • Content (subset of data)
    • Total number of pages
    • Total number of elements
    • Current page metadata (like number, size)

Pageable

  • Interface that provides pagination info such as:
    • Page number
    • Page size
    • Sorting options

PageRequest

  • A concrete implementation of the Pageable interface.
  • Used to create pagination + sorting data.
  • Example:
Pageable pageable = PageRequest.of(pageNumber, pageSize, Sort.by("price"));

Pagination and Sorting in Spring Data JPA​

Using findAll() with Pagination and Sorting​

  • You can use the findAll() method from your ProductRepository in two ways:
    • Without arguments: returns a List<Product>
    • With Pageable: returns a Page<Product>

Creating a Pageable Object​

  • To implement pagination:
Pageable pageable = PageRequest.of(pageNumber, pageSize);
  • pageNumber: Zero-based index (0 for first page)
  • pageSize: Number of records per page

Why Use Pagination?​

  • Helps avoid sending too much data to the frontend
  • Reduces bandwidth usage and server load
  • Essential for large datasets

Hardcoding pageSize​

private static final int PAGE_SIZE = 5;
  • Best practice to restrict frontend from deciding pageSize to maintain backend control.

Enhancing Pagination with Sorting​

  • You can also include sorting logic in the PageRequest:
Pageable pageable = PageRequest.of(pageNumber, pageSize, Sort.by(sortBy));
  • Sort can be customized dynamically using request parameters like sortBy.

Extracting Data from Page<T>​

  • Page<T> provides both data and metadata:
    • getContent() – Returns the actual list of data
    • getTotalPages() – Total number of pages
    • getTotalElements() – Total records
    • getNumber() – Current page number
    • getSize() – Page size
    • hasNext() / hasPrevious() – For navigation
    • isFirst() / isLast() – Position info
  • Example:
List<Product> products = productPage.getContent();
int totalPages = productPage.getTotalPages();

Controller Example​

@GetMapping("/products")
public List<Product> getProducts(
    @RequestParam(defaultValue = "0") int page,
    @RequestParam(defaultValue = "quantity") String sortBy
) {
    int pageSize = 5;
    Pageable pageable = PageRequest.of(page, pageSize, Sort.by(sortBy));
    Page<Product> productPage = productRepository.findAll(pageable);
    return productPage.getContent();
}

How Pagination Works in Practice- Page 0: Items 1–5​

  • Page 1: Items 6–10
  • Page 2: Items 11–15
  • If a non-existent page (e.g., page 20) is requested, it returns an empty list.

Combining Pagination, Sorting, and Filtering in Spring Data JPA​

Pagination with Filtered Queries​

  • You can use pagination in custom query methods (e.g., findByTitleContainingIgnoreCase(String title, Pageable pageable)).
  • This allows you to:
    • Filter records based on the title.
    • Paginate the filtered results.
    • Sort them using Sort object.

Example Repository Method​

List<ProductEntity> findByTitleContainingIgnoreCase(String chOco, Pageable pageable);
  • Accepts a title and a Pageable object.
  • Returns a paginated list of products whose title contains the given string (case-insensitive).

Controller Usage Example​

@GetMapping("all-products")
    public List<ProductEntity> getAllProducts(
            @RequestParam(defaultValue = "") String title,
            @RequestParam(defaultValue = "id") String sortBy,
            @RequestParam(defaultValue = "0") Integer pageNumber){

        return productRepository.findByTitleContainingIgnoreCase(
                title, PageRequest.of(pageNumber, PAGE_SIZE, Sort.by(sortBy))
        );
    }

Accepts query parameters for:​

  • title (filtering)
  • page (pagination)
  • sortBy (sorting field)
  • Uses PageRequest.of(page, pageSize, sort) to create a pageable instance.
  • Returns a paginated, filtered, and sorted list of products.

Output Behavior​

  • If title = "b", you may get results like Biscuit, Bonvita, etc.
  • If title = "bis", results are further narrowed to Biscuit, etc.
  • Pagination behaves as expected:
    • Page 0: items 1–5
    • Page 1: items 6–10
    • And so on.
  • Sorting works correctly, e.g., by quantity in ascending order.

Real-World Use Case​

  • This is similar to e-commerce filtering:
    • Search by keyword
    • Paginate results (e.g., 5 per page)
    • Sort by price, name, quantity, etc.

import lombok.AllArgsConstructor;
import lombok.Builder;
import lombok.Data;
import lombok.NoArgsConstructor;

import org.hibernate.annotations.UpdateTimestamp;

@Data
@AllArgsConstructor
@NoArgsConstructor
@Builder
@Entity
@Table(
        name = "product_table",
        uniqueConstraints = {
                @UniqueConstraint(name = "title_price_unique", columnNames = {"title_x", "price"})

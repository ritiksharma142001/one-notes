Understanding SQL Relational Operators with Examples

Why Relational Operators?​

Relational operators allow for complex queries involving conditions and comparisons, enabling efficient data retrieval and filtering. They're especially useful in databases with large datasets.

Practical Examples Using Relational Operators​

Scenario 1: Finding High-Cost Courses​

To identify courses with fees greater than ₹75,000:

SELECT course_name 
FROM course 
WHERE course_fees > 75000;

Output:

AI and ML

Scenario 2: Identifying Courses by Specific Fee Criteria​

To find courses where the fee is exactly ₹70,000:

SELECT course_name 
FROM course 
WHERE course_fees = 70000;

Output:

To list courses that charge more than ₹65,000:

SELECT course_name

Precedence of Relational Operators​

Understanding operator precedence is essential for writing complex SQL queries. In SQL, relational operators are evaluated in the following order (from highest to lowest precedence):

  1. = (equals)
  2. <> or != (not equals)
  3. > and < (greater than and less than)
  4. >= and <= (greater than or equal to, less than or equal to)

Operators with higher precedence are evaluated first.

Example: Using Precedence in a Query​

Suppose you want to find courses that either cost less than ₹50,000 or have a duration longer than 12 months. You prioritize checking the cost condition first.

SELECT course_name 
FROM course 
WHERE course_fees < 50000 OR duration > 12;

How This Works:​

  • Relational Operators (<, >) are evaluated first.
  • Then the Logical Operator (OR) combines the results.
  • If either condition is true, the course will be included in the output.

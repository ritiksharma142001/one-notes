Introduction to Operators

Understanding SQL Arithmetic Operators with Examples

We'll use a table called Employees with the following data to explore how SQL arithmetic operators work. Whenever you read a query or see its output, refer to this table for better understanding.

Why Are Operators Important?​

We've already seen how to set up tables and add data using concepts like primary keys and foreign keys. But to manipulate and analyze data effectively, SQL operators are crucial. They let us perform calculations, apply conditions, and retrieve data intelligently.

Types of SQL Operators​

SQL operators fall into two main categories:

1. Symbolic Operators​

Used for arithmetic and comparison.

➤ Arithmetic Operators:​

OperatorDescription
+Addition
-Subtraction
*Multiplication
/Division
%Modulo (Remainder)

➤ Comparison Operators:​

OperatorMeaning
=Equal to
>Greater than
<Less than
>=Greater than or equal to
<=Less than or equal to
!= or <>Not equal to

2. Keyword Operators​

Used for logic and filtering.

OperatorUsage
ANDCombines multiple conditions
ORAt least one condition is true
NOTNegates a condition
BETWEENFilters a range
LIKEPattern matching
INMatches any value in a list

Practical Use Cases of Arithmetic Operators​

Let’s explore real examples using our Employees table:

Scenario 1: Incrementing Employee Salaries​

SELECT Name, (Employee_Salary + 10000) AS Updated_Salary 
FROM employees;

Scenario 2: Decrementing Employee Salaries​

SELECT Name, (Employee_Salary - 5000) AS Decreased_Salary 
FROM employees;

Scenario 3: Calculating Annual Salaries​

SELECT Name, (Employee_Salary * 12) AS Annual_Salary 
FROM employees;

Scenario 4: Conditional Salary Increases​

SELECT Name, Employee_Salary * 1.10 AS Adjusted_Salary 
FROM employees 
WHERE Employee_Salary < 50000;

Scenario 5: Post-Tax Net Salaries​

SELECT Name, (Employee_Salary * 0.80) AS Net_Salary 
FROM employees;

Scenario 6: Checking Salary Divisibility​

SELECT Name, Employee_Salary 
FROM employees 
WHERE Employee_Salary % 5000 = 0;

Scenario 7: Calculating Overtime Pay​

SELECT Name, (Hourly_Rate * 1.5) AS Overtime_Rate 
FROM employees 
WHERE Hours_Worked > 40;

Scenario 8: Pro-rating Salary for Partial Months​

SELECT Name, (Employee_Salary / 30 * Hours_Worked / 8) AS Prorated_Salary 
FROM employees;

Understanding Operator Precedence​

Just like in mathematics, SQL follows operator precedence:

  1. Multiplication (*) and Division (/) are evaluated first.
  2. Addition (+) and Subtraction (-) follow.

Example:

SELECT Name, (Employee_Salary * 12 / 2) AS Semi_Annual_Salary 
FROM employees;

This query multiplies the salary by 12 and then divides the result by 2 to get the semi-annual salary.

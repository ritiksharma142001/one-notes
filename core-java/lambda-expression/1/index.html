Java Lambda Expressions

GFG

What is a Lambda Expression in Java?​

  • A lambda expression is a shorter way to write code that you normally write using an interface with one method (called a functional interface).

Example Without Lambda (Old Way):

  • Suppose you want to run some code in a new thread:
public class LamdaExpression {
    public static void main(String[] args) {
        Runnable runner = new Runnable(){
            @Override
            public void run(){
                System.out.println("Hello from a thread!");
            }
        };
        new Thread(runner).start();
    }
}

Here:

  • Runnable is a functional interface (it has only one method: run()).
  • We created an anonymous class that overrides run().

Same Example With Lambda (New Way):

Runnable r = () -> System.out.println("Hello from a thread!");
new Thread(r).start();

What is a Functional Interface?​

A functional interface is just an interface with one abstract method.

@FunctionalInterface
interface MyPrinter {
    void print(String message);
}

Using lambda:

MyPrinter printer = msg -> System.out.println(msg);
printer.print("Hello!");

Key Points:​

  • Introduced in Java 8.
  • Used to implement functional interfaces.
  • Makes code shorter, cleaner, and more readable.
  • Often used with Streams, Collections, and threads.

Key Functionalities of Lambda Expression​

  1. Functional Interfaces Lambda expressions work only with interfaces that have just one abstract method.
@FunctionalInterface
interface Greet {
    void sayHello();
}

Greet g = () -> System.out.println("Hello!");
g.sayHello();  // Output: Hello!
  1. Code as Data You can pass logic (code) as a value into methods — like passing numbers or strings.
interface MathOperation {
    int operation(int a, int b);
}

public class Main {
    static void calculate(MathOperation op, int a, int b) {
        System.out.println(op.operation(a, b));
    }

    public static void main(String[] args) {
        calculate((x, y) -> x + y, 5, 3);  // Output: 8
    }
}

Here, we pass code (x, y) -> x + y as an argument.

  1. Class Independence You don’t need to create a separate class to write a small function.

Before Lambda (Old Way):

class MyPrinterImpl implements MyPrinter {
    public void print(String msg) {
        System.out.println(msg);
    }
}

With Lambda (New Way):

MyPrinter printer = msg -> System.out.println(msg);

No class needed — just one line!

  1. Pass and Execute You can store a lambda expression in a variable and run it whenever you want.

Example:

Runnable task = () -> System.out.println("Task executed!");
task.run();  // Output: Task executed!

You can pass it around like a normal object and call it later.

Structure of Lambda Expression​

Syntax of Lambda Expressions​

(argument list) -> { body of the expression }

Components:

  1. Argument List: Parameters for the lambda expression
  2. Arrow Token (->): Separates the parameter list and the body
  3. Body: Logic to be executed.

Types of Lambda Parameters​

There are three Lambda Expression Parameters are mentioned below:

  1. Lambda with Zero Parameter

  2. Lambda with Single Parameter

  3. Lambda with Multiple Parameters

  4. Lambda with Zero Parameter

Syntax:

() -> System.out.println(“Zero parameter lambda”);

// Java program to demonstrates Lambda expression with zero parameter
@FunctionalInterface
interface ZeroParameter {
    void display();
}

public class Geeks {
    public static void main(String[] args)
    {
        // Lambda expression with zero parameters
        ZeroParameter zeroParamLambda = ()
            -> System.out.println(
                "This is a zero-parameter lambda expression!");

        // Invoke the method
        zeroParamLambda.display();
    }
}

// This is a zero-parameter lambda expression!
  1. Lambda with a Single Parameter

Syntax: (p) -> System.out.println(“One parameter: ” + p);

✅ Rule: You can skip parentheses if:

  1. There is only one parameter
  2. Java can guess the type from the functional interface

Consumer<String> c = s -> System.out.println(s);

❌ But if there are two or more parameters, you must use parentheses:

BiConsumer<String, Integer> bc = (name, age) -> System.out.println(name + " is " + age);

// Java program to demonstrate simple lambda expressions
import java.util.ArrayList;

class Test {

    public static void main(String args[])
    {
        // Creating an ArrayList with elements
        // {1, 2, 3, 4}
        ArrayList<Integer> al = new ArrayList<Integer>();
        al.add(1);
        al.add(2);
        al.add(3);
        al.add(4);

        // Using lambda expression to print all elements of al
        System.out.println("Elements of the ArrayList: ");
        al.forEach(n -> System.out.println(n));

        // Using lambda expression to print even elements
        // of al
        System.out.println(
            "Even elements of the ArrayList: ");
        al.forEach(n -> {
            if (n % 2 == 0)
                System.out.println(n);
        });
    }
}

// Elements of the ArrayList: 
// 1
// 2
// 3
// 4
// Even elements of the ArrayList: 
// 2
// 4
  1. Lambda Expression with Multiple Parameters

Syntax: (p1, p2) -> System.out.println(“Multiple parameters: ” + p1 + “, ” + p2);

@FunctionalInterface
interface Functional {
    int operation(int a, int b);
}

public class Test {

    public static void main(String[] args) {
        
        // Using lambda expressions to define the operations
        Functional add = (a, b) -> a + b;
        Functional multiply = (a, b) -> a * b;

        // Using the operations
        System.out.println(add.operation(6, 3));  
        System.out.println(multiply.operation(4, 5));  
    }
}

// 9
// 20

Commonly Used Functional Interfaces in Java​

  1. Predicate<T> Checks a condition and returns true or false.

Example:

Predicate<String> checkEmpty = str -> str.isEmpty();
  1. Function<T, R>

Takes an input of type T, returns a result of type R.

Example:

Function<Integer, String> convert = i -> "Value: " + i;
  1. Supplier<T>

Takes no input, but returns a value of type T.

Example:

Supplier<Double> randomSupplier = () -> Math.random();
  1. Consumer<T>

Takes an input and returns nothing (void). Example:

Consumer<String> print = s -> System.out.println(s);

Lambda Expression Rules​

  1. Lambda Must Be Assigned to a Functional Interface A lambda expression can only be used where a functional interface is expected.

✅ Example:

Runnable r = () -> System.out.println("Hello");

❌ Invalid:

Object o = () -> System.out.println("Invalid"); // Error: Object is not a functional interface
  1. Parameter Rules

✅ One parameter — parentheses optional:

Consumer<String> c = s -> System.out.println(s);  // valid

✅ One parameter — parentheses allowed:

Consumer<String> c = (s) -> System.out.println(s);  // valid

✅ Two or more parameters — parentheses required:

BiConsumer<String, Integer> bc = (name, age) -> System.out.println(name + " is " + age);

❌ Invalid — missing parentheses with two parameters:

BiConsumer<String, Integer> bc = name, age -> System.out.println(name); // ❌ Error
  1. Type Declaration in Parameters ✅ Optional (type is inferred):
Predicate<Integer> isEven = n -> n % 2 == 0;

✅ Allowed (explicit type):

Predicate<Integer> isEven = (Integer n) -> n % 2 == 0;

❌ Mixed: you can't mix typed and untyped parameters:

BiFunction<Integer, Integer, Integer> sum = (a, Integer b) -> a + b; // ❌ Invalid
  1. Body Rules ✅ Single expression — no braces, no return:
Supplier<String> s = () -> "Hello";

✅ Multiple statements or use of return — braces are required:

Supplier<String> s = () -> {
    String greeting = "Hello";
    return greeting;
};

❌ Using return without braces:

Supplier<String> s = () -> return "Hello"; // ❌ Error
  1. Void-returning lambdas (like Runnable) can skip return
Runnable r = () -> System.out.println("Running");
  1. Semicolon Misuse ❌ Don't add a semicolon outside the lambda body block.
Function<String, String> f = (s) -> { return "Hi"; };  // ✔ VALID
Function<String, String> f = (s) -> { return "Hi"; };  // ✔ ALSO VALID

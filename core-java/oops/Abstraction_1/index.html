    abstract void project();
}

    void teach(){
        System.out.println("Teaching Java");
    }
    void project(){
        System.out.println("Java Project");
    }
}
class PythonTeacher extends Teacher{
    void teach(){
        System.out.println("Teaching Python");
    }
    void project(){
        System.out.println("Python Project");
    }
}

public class Main {
    public static void teacherBehaviour(Teacher t){
        t.teach();
        t.project();
    }
    public static void main(String[] args) {
        JavaTeacher j = new JavaTeacher();
        PythonTeacher p = new PythonTeacher();
        teacherBehaviour(j);
        teacherBehaviour(p);
    }
}

Constructor Chaining in Java with Abstract Classes

The reason the constructor of Teacher is called, even though you cannot create an object of the Teacher class directly, is because JavaTeacher is a subclass of Teacher. When you create an object of JavaTeacher, it still inherits the constructor of Teacher, and the constructor of the superclass (Teacher) is called as part of the object creation process.

1. Object Creation and Constructor Chaining

When you create an object of a subclass (in this case, JavaTeacher), the constructor of the parent class (Teacher) is always called first before the constructor of the subclass (JavaTeacher). This is called constructor chaining.

Even though you can't instantiate the Teacher class directly, the subclass (JavaTeacher) still needs to initialize the parent class (Teacher), and this is done by calling the constructor of the parent class.

2. How Constructor Chaining Works

  • When the JavaTeacher constructor is invoked, the default constructor of Teacher (the parent class) is automatically called first, unless you explicitly call a specific constructor of Teacher using super().

  • In your code, you did not explicitly call super(), so the default constructor of Teacher (which prints "Constructor of Teacher") is called automatically before the constructor of JavaTeacher.

3. You Can't Instantiate Teacher Directly, But It's Still Involved

Even though you can't instantiate Teacher directly because it's abstract, Java still calls its constructor as part of the object creation process of the JavaTeacher class.

The constructor of Teacher is executed to ensure the proper initialization of any inherited fields or methods from Teacher that the JavaTeacher class might use.

Here's the Flow of Execution

  1. Static block of Teacher is executed (since Teacher is loaded when JavaTeacher is instantiated).
  2. Static block of JavaTeacher is executed (since JavaTeacher is now being loaded).
  3. Instance block of Teacher is executed (instance initialization for the superclass).
  4. Constructor of Teacher is executed (initializing the superclass part of the object).
  5. Instance block of JavaTeacher is executed (instance initialization for the subclass).
  6. Constructor of JavaTeacher is executed (initializing the subclass part of the object).
  7. Finally, the teach() method of JavaTeacher is called.

Understanding Static Block Execution Order in Java

In Java, static blocks are executed when a class is loaded into memory for the first time. When you create an object of a class (e.g., JavaTeacher obj = new JavaTeacher();), both the parent class (Teacher) and the child class (JavaTeacher) are loaded into memory.

Why Is the Static Block of Teacher Called?

  • Static blocks execute when the class is loaded, regardless of whether an object is created or not.
  • When you create an object of JavaTeacher, both Teacher (its superclass) and JavaTeacher (the subclass) are loaded into memory.

Class Loading Order

  1. When the JavaTeacher class is loaded for the first time, the Teacher class (its superclass) is loaded first because JavaTeacher extends Teacher.
  2. As part of loading the Teacher class, the static blocks in Teacher are executed.
  3. After the Teacher static block completes, the static block of JavaTeacher is executed.

Execution Order When You Run the Code

  1. Static block of Teacher is executed first
    (because Teacher is the superclass and is loaded first).
  2. Static block of JavaTeacher is executed next
    (because JavaTeacher is now being loaded).
  3. Instance block of Teacher is executed
    (instance initialization of the superclass).
  4. Constructor of Teacher is executed
    (initializing the superclass part of the object).
  5. Instance block of JavaTeacher is executed
    (instance initialization of the subclass).
  6. Constructor of JavaTeacher is executed
    (initializing the subclass part of the object).

interface StudentActivities {
    void study(); // abstract void study();
}
class PhysicsStudent implements StudentActivities {
    @Override
    public void study() {
        System.out.println("Studying Physics");
    }
}
class MathStudent implements StudentActivities {
    @Override
    public void study() {
        System.out.println("Studying Math");
    }
}
public class Main {
    static void study(StudentActivities studentActivities) {
        studentActivities.study();
    }
    public static void main(String[] args) {
       PhysicsStudent physicsStudent = new PhysicsStudent();
       MathStudent mathStudent = new MathStudent();
       study(physicsStudent);
       study(mathStudent);
    }
}

interface cal1{
    public abstract void disp1();
    public default void disp2(){
        System.out.println("Default Method");
    }
    public static void disp3(){
        System.out.println("Static Method");
    }
    private void disp4(){
        System.out.println("Private Method"); // not accessible outside interface
    }
}
class cal2 implements cal1{
    public void disp1(){
        System.out.println("Abstract Method");
    }
    public void disp2(){
        System.out.println("Default Method Overriden");
    }
    public void disp4(){
        System.out.println("Private Method Overriden");
    }
    public static void disp3(){
        System.out.println("Static Method Overriden"); //this does not override static method
        // just the another method with same name
    }
}
public class Main {
    public static void main(String[] args) {
        // cal1 obj = new cal2();
        // obj.disp1();
        // obj.disp2();
        // cal1.disp3();
        // // print static method overridden
        // cal2.disp3();
        // // print private method overridden
        // // obj.disp4();

        cal1 obj = new cal2();
        obj.disp1();  // Calls the abstract method (overridden in cal2)

        obj.disp2();  // Calls the default method (overridden in cal2)

        // Calling the static method of cal1 (from the interface)
        cal1.disp3();  // Output: Static Method

        // Calling the static method of cal2 (overrides the static method of cal1)
        cal2.disp3();  // Output: Static Method Overridden

        // Cannot access private methods from the interface
        // obj.disp4();  // This will cause a compile-time error because disp4 is private


    } 
}

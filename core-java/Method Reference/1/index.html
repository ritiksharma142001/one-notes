Method Reference in Java (Java 8)‚Äã

A method reference in Java is a shorthand way of referring to a method without executing it. It is used with lambda expressions to improve readability.

Syntax of Method Reference‚Äã

  • It is used when a lambda expression calls an existing method.
  • It makes code shorter and more readable.

Types of Method References‚Äã

There are four types of method references:

1Ô∏è‚É£ Reference to a Static Method‚Äã

import java.util.function.Function;

public class StaticMethodReference {
    // Static method
    public static int square(int num) {
        return num * num;
    }

    public static void main(String[] args) {
        // Using Lambda Expression
        Function<Integer, Integer> lambdaSquare = num -> StaticMethodReference.square(num);

        // Using Method Reference
        Function<Integer, Integer> methodRefSquare = StaticMethodReference::square;

        System.out.println(methodRefSquare.apply(5)); // Output: 25
    }
}

4 Reference to a Constructor‚Äã

ClassName::new
import java.util.function.Supplier;

class Car {
    Car() {
        System.out.println("Car is created!");
    }
}

public class ConstructorReference {
    public static void main(String[] args) {
        // Using Lambda Expression
        Supplier<Car> lambdaCar = () -> new Car();

        // Using Constructor Reference
        Supplier<Car> methodRefCar = Car::new;

        // Creates a new Car object
        methodRefCar.get();
    }
}

// Car is created!

Why Use Method References?‚Äã

import java.util.function.Function;

class Car {
    private String brand;

    // Parameterized Constructor
    public Car(String brand) {
        this.brand = brand;
        System.out.println("Car brand: " + brand);
    }
}

public class ConstructorReferenceExample {
    public static void main(String[] args) {
        // Using Lambda Expression
        Function<String, Car> lambdaCar = brand -> new Car(brand);

        // Using Constructor Reference
        Function<String, Car> methodRefCar = Car::new;

        // Creating an object
        methodRefCar.apply("Toyota"); // Output: Car brand: Toyota
    }
}

üìù Example 2: Constructor with Two Parameters

import java.util.function.BiFunction;

class Bike {
    private String brand;
    private int price;

    // Parameterized Constructor
    public Bike(String brand, int price) {
        this.brand = brand;
        this.price = price;
        System.out.println("Bike brand: " + brand + ", Price: " + price);
    }
}

public class ConstructorReferenceExample {
    public static void main(String[] args) {
        // Using Lambda Expression
        BiFunction<String, Integer, Bike> lambdaBike = (brand, price) -> new Bike(brand, price);

        // Using Constructor Reference
        BiFunction<String, Integer, Bike> methodRefBike = Bike::new;

        // Creating an object
        methodRefBike.apply("Yamaha", 100000); // Output: Bike brand: Yamaha, Price: 100000
    }
}

üìù Example 3: Constructor with Multiple Parameters and Custom Functional Interface If more than two parameters are required, create a custom functional interface.

import java.util.function.Function;

// Functional Interface for three parameters
@FunctionalInterface
interface TriFunction<T, U, V, R> {
    R apply(T t, U u, V v);
}

class Person {
    private String name;
    private int age;
    private String city;

    // Parameterized Constructor
    public Person(String name, int age, String city) {
        this.name = name;
        this.age = age;
        this.city = city;
        System.out.println("Person: " + name + ", Age: " + age + ", City: " + city);
    }
}

public class ConstructorReferenceExample {
    public static void main(String[] args) {
        // Using Lambda Expression
        TriFunction<String, Integer, String, Person> lambdaPerson = (name, age, city) -> new Person(name, age, city);

        // Using Constructor Reference
        TriFunction<String, Integer, String, Person> methodRefPerson = Person::new;

        // Creating an object
        methodRefPerson.apply("John", 25, "New York");
    }
}

Key Benefits of Method References Improved Readability: Method references simplify the code by removing boilerplate syntax. Reusability: Existing methods can be directly reused, enhancing modularity. Functional Programming Support: They work seamlessly with functional interfaces and lambdas.

Converting ArrayList to HashMap using Method Reference in Java 8 üìù Simple Explanation: ‚Ä¢ We have an ArrayList of Strings. ‚Ä¢ We need to convert it into a HashMap where: ‚óã The key is the element itself. ‚óã The value is the length of the string. Method Reference (Function.identity() and String::length) makes the code cleaner.

import java.util.*;
import java.util.function.Function;
import java.util.stream.Collectors;

public class Main {
    public static void main(String[] args) {
        // Create an ArrayList of Strings
        List<String> fruits = Arrays.asList("Banana", "Guava", "Pineapple", "Apple");

        // Convert ArrayList to HashMap using Method Reference
        Map<String, Integer> fruitMap = fruits.stream()
                .collect(Collectors.toMap(
                        Function.identity(),  // Key -> element itself
                        String::length        // Value -> length of the string
                ));

        // Print the HashMap
        System.out.println(fruitMap);
    }
}

üìù Explanation of collect(Collectors.toMap(...)) ‚Ä¢ Function.identity() ‚Üí Uses the element itself as the key. ‚Ä¢ String::length ‚Üí Uses the length of the string as the value. üñ• Output:

{Guava=5, Apple=5, Pineapple=9, Banana=6}

Functional Interface

An interface that contains only one abstract method is known as a functional interface, but there is no restriction, you can have n number of default and static methods inside a functional interface.

Creating Threads in Java: Anonymous Class vs. Lambda Expression‚Äã

‚úÖ Using Anonymous Class‚Äã

public class Geeks {
    public static void main(String[] args) {
        Runnable r = new Runnable() {
            @Override
            public void run() {
                System.out.println("New thread created");
            }
        };

        Thread t = new Thread(r);
        t.start();
    }
}

Using Lambda Expression‚Äã

public class Geeks {
    public static void main(String[] args) {
        // Using lambda expression 
        // to implement Runnable
        new Thread(() -> System.out.println("New thread created")).start();
    }
}

Explanation‚Äã

1Ô∏è‚É£ new Thread(...) ‚Ä¢ This creates a new thread.

2Ô∏è‚É£ () -> System.out.println("New thread created") ‚Ä¢ This is a lambda expression that implements Runnable (which requires a run() method). ‚Ä¢ Instead of writing a full anonymous class, we use a shorter lambda.

3Ô∏è‚É£ .start() ‚Ä¢ This starts the thread and executes the code inside run().

‚Ä¢ Here, we use an anonymous class to define Runnable. ‚Ä¢ Lambda expressions make the code shorter and cleaner.

üßæ Java Program to Demonstrate Functional Interface (Before Java 8)‚Äã

// Java program to demonstrate functional interface
// before Java 8
class Geeks {
    public static void main(String args[]) {
        // create anonymous inner class object
        new Thread(new Runnable() {
            @Override public void run() {
                System.out.println("New thread created");
            }
        }).start();
    }
}

What Happens Internally?‚Äã

A new thread is created.

The thread executes the lambda expression (which prints "New thread created").

The main thread and the new thread run independently.

@FunctionalInterface Annotation‚Äã

@FunctionalInterface annotation is used to ensure that the functional interface cannot have more than one abstract method.

üîπ If more than one abstract method is present, the compiler flags an:

"Unexpected @FunctionalInterface annotation" message.

üîπ However, it is not mandatory to use this annotation.

// Define a functional interface
@FunctionalInterface

interface Square {
    int calculate(int x);
}

class Geeks {
    public static void main(String args[]) {
        int a = 5;

        // lambda expression to 
        // define the calculate method
        Square s = (int x) -> x * x;

        // parameter passed and return type must be
        // same as defined in the prototype
        int ans = s.calculate(a);
        System.out.println(ans); // 25
    }
}

üîπ 1. Consumer<T>‚Äã

  • Purpose: Takes an input but does not return a value.
  • Common Use: Performing operations such as printing or modifying an object.
  1. Using Anonymous Class
import java.util.function.Consumer;

public class Main {
    public static void main(String[] args) {
        Consumer<String> printName = new Consumer<String>() {
            @Override
            public void accept(String name) {
                System.out.println("Hello, " + name);
            }
        };

        printName.accept("Alice"); // Calls Consumer
    }
}
  1. Using Lambda Expression
import java.util.function.Consumer;

public class Main {
    public static void main(String[] args) {
        Consumer<String> printName = name -> System.out.println("Hello, " + name);

        printName.accept("Alice"); // Calls Consumer
    }
}

‚úÖ Use Case: Logging, printing, modifying objects.

üîπ 2. Predicate<T>‚Äã

  • Purpose: Takes an input and returns a boolean (true or false).
  • Common Use: Filtering data or performing conditional checks.
import java.util.function.Predicate;

public class Main {
    public static void main(String[] args) {
        Predicate<Integer> isEven = new Predicate<Integer>() {
            @Override
            public boolean test(Integer num) {
                return num % 2 == 0;
            }
        };

        System.out.println(isEven.test(4)); // true
        System.out.println(isEven.test(7)); // false
    }
}
import java.util.function.Predicate;

public class Main {
    public static void main(String[] args) {
        Predicate<Integer> isEven = num -> num % 2 == 0;

        System.out.println(isEven.test(4)); // true
        System.out.println(isEven.test(7)); // false
    }
}

‚úÖ Use Case: Filtering lists, validating input, checking conditions.

üîπ 3. Function<T, R>‚Äã

  • Purpose: Takes an input of type T and returns a result of type R.
  • Common Use: Data transformation or mapping operations.
import java.util.function.Function;

public class Main {
    public static void main(String[] args) {
        Function<Integer, String> convertToString = new Function<Integer, String>() {
            @Override
            public String apply(Integer num) {
                return "Number: " + num;
            }
        };

        System.out.println(convertToString.apply(5));
    }
}
import java.util.function.Function;

public class Main {
    public static void main(String[] args) {
        Function<Integer, String> convertToString = num -> "Number: " + num;

        System.out.println(convertToString.apply(5));
    }
}

‚úÖ Use Case: Converting data types, transforming objects, calculations.

üîπ 4. Supplier<T>‚Äã

  • Purpose: Takes no input but returns a value of type T.
  • Common Use: Supplying values, such as lazy initialization or generating random values.
import java.util.function.Supplier;

public class Main {
    public static void main(String[] args) {
        Supplier<Double> getRandomNumber = new Supplier<Double>() {
            @Override
            public Double get() {
                return Math.random();
            }
        };

        System.out.println(getRandomNumber.get());
    }
}
import java.util.function.Supplier;

public class Main {
    public static void main(String[] args) {
        Supplier<Double> getRandomNumber = () -> Math.random();

        System.out.println(getRandomNumber.get());
    }
}

‚úÖ Use Case: Generating random numbers, getting current time, providing default values.

Important Points about Functional Interfaces in Java‚Äã

  • Single Abstract Method: A functional interface can have only one abstract method. If more than one abstract method is declared, it becomes a non-functional interface.

  • @FunctionalInterface Annotation is Optional:

    • It is not mandatory to use @FunctionalInterface.
    • However, using it allows the compiler to enforce that the interface has only one abstract method.

  • Unlimited Static and Default Methods:

    • A functional interface can contain any number of static and default methods.
    • These do not affect its status as a functional interface.

  • Overriding Parent Methods:

    • Overriding methods from interfaces or classes (like Object.toString()) does not violate the functional interface rule.

  • Built-in Functional Interfaces:

    • Java 8 introduced several functional interfaces in the java.util.function package such as:
      • Predicate<T>
      • Consumer<T>
      • Function<T, R>
      • Supplier<T>
      • And more...

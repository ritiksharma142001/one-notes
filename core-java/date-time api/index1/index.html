Date/Time API

Reference taken from GFG - GeeksForGeeks Link for Date/Time API

Why Java 8 Introduced a New Date-Time API?‚Äã

The old Date API (java.util.Date, Calendar, etc.) had some problems:

Problems with Old Date API:

  1. Not thread-safe:
  • If multiple threads use Date, it can lead to bugs.
  • You need to manually add synchronization.
  1. Limited operations:
  • Very few built-in methods for working with dates (like adding days, formatting, etc.).
  1. Complex and confusing:
  • APIs like Calendar were hard to use and understand.

What Java 8 Offers (java.time package):‚Äã

Java 8 introduced a new and improved Date-Time API in the java.time package.

Key Features:

  1. Immutable and Thread-safe:
  • You can't change the date object after it's created (just like String). Safe for multi-threaded use.
  1. Easy to use and powerful:
  • Many methods for adding, subtracting, formatting, and comparing dates.

Important Classes in Java 8 Date-Time API‚Äã

1. Local API (No Timezone)‚Äã

  • LocalDate ‚Äì Represents only the date
    Example: 2025-05-10

  • LocalTime ‚Äì Represents only the time
    Example: 14:30:00

  • LocalDateTime ‚Äì Represents both date and time
    Example: 2025-05-10T14:30

Use these when you don‚Äôt need to care about time zones.

2. Zoned API (With Timezone)‚Äã

  • ZonedDateTime ‚Äì Includes date, time, and timezone
    Example: 2025-05-10T14:30+05:30[Asia/Kolkata]

Use this when you need to work with time zones.

Summary: Old API vs New API‚Äã

FeatureOld API (Date, Calendar)New API (java.time)
Thread-safe‚ùå No‚úÖ Yes
Easy to use‚ùå Confusing‚úÖ Simple
Timezone support‚ùå Difficult‚úÖ Clean (Zoned classes)
Immutable‚ùå No‚úÖ Yes

LocalDate:‚Äã

LocalDate date = LocalDate.now();

  • A Java 8 class from the java.time package that represents a date without a time or timezone, such as 2025-05-10.

  • LocalDate.now(): This is a static method that returns the current date from the system clock, in the default timezone.

  • date: A variable of type LocalDate that stores the current date.

# üìÖ Java 8 Date-Time API Example

This Java program demonstrates how to use `LocalDate`, `LocalTime`, `LocalDateTime`, and `DateTimeFormatter` from the `java.time` package introduced in Java 8.

---

## ‚úÖ Code:

```java
import java.time.LocalDate;
import java.time.LocalDateTime;
import java.time.LocalTime;
import java.time.format.DateTimeFormatter;

public class Date {
    public static void main(String[] args) {

        // Current Date
        System.out.println(LocalDate.now());

        // Current Time
        System.out.println(LocalTime.now());

        // Current Date & Time
        System.out.println(LocalDateTime.now());

        // Custom formatted current Date & Time
        System.out.println(LocalDateTime.now().format(
            DateTimeFormatter.ofPattern("hh:ss:mm MM-dd-yyyy")
        ));

        // Get current month
        System.out.println(LocalDate.now().getMonth());

        // Get day of month
        System.out.println(LocalDate.now().getDayOfMonth());

        // Get day of week
        System.out.println(LocalDate.now().getDayOfWeek());

        // Get day of year
        System.out.println(LocalDate.now().getDayOfYear());

        // Get seconds from current time
        System.out.println(LocalDateTime.now().getSecond());

        // Create a specific date: May 21, 3022
        System.out.println(LocalDate.of(3022, 5, 21));

        // Create a specific date and time: April 12, 2027 at 03:45:23
        System.out.println(LocalDateTime.of(2027, 4, 12, 3, 45, 23));


        System.out.println(LocalDateTime.now());
        System.out.println(ZonedDateTime.now());
        System.out.println(ZonedDateTime.now().getZone());
        System.out.println(ZoneId.of("Asia/Tokyo"));
    }
}

checkingPeriod() ‚Äî Used for Date Differences‚Äã

public static void checkingPeriod() {
    LocalDate date1 = LocalDate.now(); // Today's date
    LocalDate date2 = LocalDate.of(2014, Month.DECEMBER, 12); // A specific past date

    Period gap = Period.between(date2, date1); // Calculates the difference between the two dates
    System.out.println("gap between dates is a period of " + gap);
}

What It Does:

  • date1 is set to today's date.

  • date2 is set to December 12, 2014.

  • Period.between(date2, date1) calculates the time gap between them in years, months, and days.

Example Output:

gap between dates is a period of P10Y4M28D (Which means 10 years, 4 months, 28 days ‚Äî actual values depend on today‚Äôs date.)

checkingDuration() ‚Äî Used for Time Differences‚Äã

public static void checkingDuration() {
    LocalTime time1 = LocalTime.now(); // Current time
    System.out.println("the current time is " + time1);

    Duration fiveHours = Duration.ofHours(5); // 5-hour duration

    LocalTime time2 = time1.plus(fiveHours); // Adds 5 hours to current time

    System.out.println("after adding five hours of duration " + time2);

    Duration gap = Duration.between(time2, time1); // Finds the gap between the two times
    System.out.println("duration gap between time1 & time2 is " + gap);
}

What It Does:

  • time1 stores the current time.

  • fiveHours creates a duration of 5 hours.

  • time2 = time1.plus(fiveHours) adds 5 hours to the current time.

  • Duration.between(time2, time1) calculates the difference between the two times.

Example Output:

the current time is 14:00
after adding five hours of duration 19:00
duration gap between time1 & time2 is PT-5H
(PT-5H means negative 5 hours, because time2 is after time1.)

üßæ What is ChronoUnit?‚Äã

Java 8 introduced the ChronoUnit enum in the java.time.temporal package to replace the old way of using integer constants (like 1 for days, 2 for months, etc.) when performing date/time calculations.

‚úÖ Example Code: Using ChronoUnit‚Äã

import java.time.LocalDate;
import java.time.temporal.ChronoUnit;

public class Geeksforgeeks {

    // Function to demonstrate ChronoUnit usage
    public static void checkingChronoEnum() {
        LocalDate date = LocalDate.now();
        System.out.println("Current date is: " + date);

        // Add 2 years
        LocalDate year = date.plus(2, ChronoUnit.YEARS);
        System.out.println("Next to next year is: " + year);

        // Add 1 month
        LocalDate nextMonth = date.plus(1, ChronoUnit.MONTHS);
        System.out.println("The next month is: " + nextMonth);

        // Add 1 week
        LocalDate nextWeek = date.plus(1, ChronoUnit.WEEKS);
        System.out.println("Next week is: " + nextWeek);

        // Add 2 decades (20 years)
        LocalDate decade = date.plus(2, ChronoUnit.DECADES);
        System.out.println("20 years after today: " + decade);
    }

    public static void main(String[] args) {
        checkingChronoEnum();
    }
}

üß† Explanation‚Äã

CodeWhat it does
ChronoUnit.YEARSAdds years
ChronoUnit.MONTHSAdds months
ChronoUnit.WEEKSAdds weeks
ChronoUnit.DECADESAdds decades (10 years = 1 decade)

üí° Why Use ChronoUnit?‚Äã

  • ‚úÖ Clear and readable
  • ‚úÖ Type-safe (no magic numbers)
  • ‚úÖ Works with all time units: DAYS, MONTHS, YEARS, HOURS, MINUTES, etc.

Generics

Generics in Java​

Generics were added in Java 5 to make code safer and reusable. They allow us to write classes, interfaces, and methods that can work with different data types while ensuring type safety.

Why Use Generics?​

  • Prevents Errors – Avoids ClassCastException, which happens when an object is wrongly cast to another type.
  • Type Safety – Checks types at compile time, reducing bugs.
  • Less Code Repetition – We don’t need to write the same code multiple times for different data types.
  • Reduce Code Redundancy

When Does ClassCastException Occur in Java?​

A ClassCastException happens when we try to cast an object to a class it doesn’t belong to. This is a runtime error, meaning it happens while the program is running, not during compilation.

  • Example Without Generics (Causing ClassCastException)
import java.util.ArrayList;

public class Main {
    public static void main(String[] args) {
        // Creating an ArrayList without Generics (raw type)
        ArrayList list = new ArrayList();
        list.add("Hello"); // Adding a String
        list.add(100);     // Adding an Integer

        // Trying to cast an Integer as a String (Incorrect casting)
        String str = (String) list.get(1); // This will cause ClassCastException
        System.out.println(str);
    }
}

  • We added both a String ("Hello") and an Integer (100) to the list.
  • Later, we tried to retrieve an Integer (100) but cast it as a String.
  • Since 100 is not a String, Java throws a ClassCastException.
  • Example With Generics (Preventing ClassCastException)
import java.util.ArrayList;

public class Main {
    public static void main(String[] args) {
        // Using Generics to ensure only Strings are allowed
        ArrayList<String> list = new ArrayList<>();
        list.add("Hello");
        // list.add(100); // This will give a compile-time error

        String str = list.get(0); // No need for casting
        System.out.println(str);
    }
}

How Does Generics Help?​

  • Generics allow us to specify a type (like <String>) for a collection or class, ensuring type safety.
  • For example, if we declare an ArrayList<String>, only String values can be added to the list.
  • If we try to add an incompatible type, such as an Integer, we get a compile-time error, preventing potential ClassCastException at runtime.

Advantages of Using Generics​

Generics in Java provide several benefits, making code safer and more efficient.

  1. Type Safety – Prevents errors like ClassCastException, which happens when an object is incorrectly cast to another type.
  2. Reusability – Allows writing code that works with different data types without rewriting it multiple times.
  3. No Need for Type Casting – Since the correct type is already defined, there’s no need to manually cast objects, reducing errors.

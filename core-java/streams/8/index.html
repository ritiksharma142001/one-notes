Streams Method

forEach()​

  • The forEach() method is used to perform an action on each element of the stream.
  • It’s often used to print, store, or apply some effect on each item in the stream
  • forEach() is a terminal operation – it triggers the processing of the stream.
  • It takes a Consumer, which means you give it a block of code (an action) that runs for every element.
  • It returns nothing (void) because its purpose is to do something, not to return a value.
  • It may run in any order, especially if used on parallel streams, so the result might not always be predictable.
  • forEach() goes through each element in the stream and performs the action you give it (like printing or saving it somewhere)

Syntax

void forEach(Consumer<? super T> action)

Where, Consumer is a functional interface
and T is the type of stream elements.

Example 1 : To perform print operation on each element of reversely sorted stream.

class GFG {  
    public static void main(String[] args) 
    { 
        List<Integer> list = Arrays.asList(2, 4, 6, 8, 10); 
        list
            .stream()
            .sorted(Comparator.reverseOrder())
            .forEach(System.out::println); 
    } 
}

// output
10

forEachOrdered()​

  • he forEachOrdered() method is similar to forEach(), but with one key difference:
  • It guarantees the order in which elements are processed, if the stream has an order.
  • It's a terminal operation, meaning it starts the processing of the stream.
  • It takes a Consumer (a function that does something for each element, like printing).
  • It processes elements in the same order they appear in the stream.
  • It is especially useful when working with parallel streams but you still want to preserve the original order.

Syntax

void forEachOrdered(Consumer<? super T> action)

Where, Consumer is a functional interface which 
is expected to operate via side-effects. and T 
is the type of stream elements.

Example difference:

List<String> names = List.of("Alice", "Bob", "Charlie");

// Using parallelStream with forEach (order not guaranteed)
names.parallelStream().forEach(System.out::println);

// Using parallelStream with forEachOrdered (order preserved)
names.parallelStream().forEachOrdered(System.out::println);
  • In the second case (forEachOrdered), names will always print in the original order: Alice, Bob, Charlie.

Example 1 : To print the elements of integer array in original order.

class GFG { 
      
    // Driver code 
    public static void main(String[] args) {

    List<Integer> list = Arrays.asList(10, 19, 20, 1, 2); 
      
    // Using forEachOrdered(Consumer action) to  
    // print the elements of stream in encounter order 
    list.stream().forEachOrdered(System.out::println); 
      
  
} 
}

foreach() loop vs Stream foreach() vs Parallel Stream foreach()​

  • for-each Loop in Java (without Lambda)
  • The for-each loop in Java is a basic way to go through elements in a collection or array.
  • It does not use lambda expressions, which means:
  • You’re just using a local variable to access each item one by one.
  • You can also use or change other variables outside the loop.
  • It’s mainly used for simple iteration (going through each element and doing something with it).
public class GFG {

	public static void main(String[] args)
	{
		String[] arr = { "1", "2", "3" };
		int count = 0;

		String[] arr1 = { "Geeks", "For", "Geeks" };

		for (String str : arr) {
			System.out.print(arr1[count++]);
		}
	}
}

// output
GeeksForGeeks

Return statements work inside a for-each loop

  • You can use a return statement inside a for-each loop to exit the loop and return a value as soon as a condition is met.
  • This is useful when you want to stop looping early and return something.
public static String frechlop(String[] geek) {
    int count = 0;
    for (String var : geek) {
        if (count == 1)
            return var;
        count++;
    }
    return "";
}
String[] arr1 = { "Geeks", "For", "Geeks" };
String secelt = frechlop(arr1);
System.out.println(secelt);

// output
For

stream().forEach() uses Lambda Expressions

  • When using stream().forEach(), you typically write code using a lambda like this
list.stream().forEach(item -> { ... });
  • Lambdas are self-contained, meaning:
  • You cannot modify variables outside of the lambda (like count++ in your example).
  • You can read external variables if they are effectively final, but not change them.

⚠ Example: This will cause an error

int count = 0;
list.stream().forEach(s -> {
    count++; // ❌ Not allowed
});

Only Works on Collections (and arrays with Arrays.stream) stream() is a method available for collections like List, Set, etc.

Arrays don’t have a .stream() method directly.

To use streams on arrays, you must write:

Arrays.stream(array)

⚠ Wrong:

String[] arr = {"a", "b"};
arr.stream().forEach(...); // ❌ Error: no such method

No Return Inside forEach() You cannot return a value from inside forEach() like you can in a normal loop.

forEach() is meant for side effects only (like printing or updating a database), not for returning a result.

If you want to find or extract something from a stream, use other methods like .filter(), .findFirst(), etc.

⚠ Example:

list.stream().forEach(s -> {
    if (s.equals("For")) return s; // ❌ Not allowed
});

What is parallelStream().forEach()?

  • parallelStream().forEach() is similar to stream().forEach(), but it uses multithreading.
  • This means it can process multiple elements at the same time, using different threads.
  • It is designed to be faster for large data sets or time-consuming tasks.
  • It uses lambda expressions like regular forEach().
  • It's a parallel (multi-threaded) version of stream processing.
  • The order of execution is not guaranteed — elements may be processed in any order.
  • Best used when order doesn't matter and you want to improve performance.
public class GFG {
 
    public static void main(String[] args)
        throws Exception
    {
 
        List<String> arr1
            = new ArrayList<String>();
        arr1.add("Geeks");
        arr1.add("For");
        arr1.add("Geeks");
 
        arr1
            .parallelStream()
            .forEach(
                s -> {
                    System.out.print(s);
                });
    }
}
// output
ForGeeksGeeks

Stream.of(T t)​

  • Stream.of(T t) creates a sequential stream with a single element.
  • It is used when you want to perform stream operations on just one item.
  • The method takes one parameter t (the element) and returns a stream containing only that element.

Syntax

static <T> Stream<T> of(T t)
class GFG { 
  
    // Driver code 
    public static void main(String[] args) 
    { 
        // Creating an Stream having single element only 
        Stream stream = Stream.of("Geeks"); 
  
        // Displaying the Stream having single element 
        stream.forEach(System.out::println); 
    } 
} 
// output
Geeks

Stream.of(T... values)?

  • Stream.of(T... values) is a method that creates a stream with multiple values.
  • It allows you to pass a list of values (as varargs), and it will return a sequential stream where elements are processed in the same order as given.
  • T... values: One or more values to include in the stream.
  • returned value -> T... values: One or more values to include in the stream.
static <T> Stream<T> of(T... values)
import java.util.stream.Stream;

public class GFG {
    public static void main(String[] args) {
        Stream<String> stream = Stream.of("Geeks", "for", "Geeks");
        stream.forEach(System.out::println);
    }
}
// output
Geeks

Use Case:

  • Use this when you have a fixed set of values and want to process them using Stream operations like filter(), map(), or forEach().

findAny()​

  • findAny() is a terminal operation in Java Streams.
  • It returns any one element from the stream (not guaranteed to be the first).
  • The result is wrapped in an Optional — this means the result may or may not be present.
  • If the stream is empty, it returns an empty Optional.
  • No parameters.
  • Works on the stream directly.
  • An Optional<T>:
    • If a value exists: it holds one element from the stream.
    • If the stream is empty: it holds nothing (i.e., is empty).
  • If the element returned is null, and you try to use it without checking, you might get a NullPointerException.

Syntax:

Optional<T> findAny()
import java.util.*;
import java.util.stream.*;

public class GFG {
    public static void main(String[] args) {
        List<String> list = Arrays.asList("Geeks", "for", "Geeks");

        // Creating a stream and using findAny()
        Optional<String> result = list.stream().findAny();

        // Print the result if present
        result.ifPresent(System.out::println);
    }
}
//output
Geeks

Stream.anyMatch() in Simple Terms:​

  • anyMatch() is a terminal operation in Java Streams that checks if any element in the stream matches a given condition (predicate). It short-circuits, meaning it stops processing as soon as it finds a match.
  • Purpose: It checks whether at least one element in the stream matches the condition you provide (via a predicate).
  • It stops as soon as it finds a match — this is called short-circuiting.
  • If the stream is empty, it returns false without evaluating the predicate.
  • true if at least one element matches the predicate.
  • false if no elements match or the stream is empty.
boolean anyMatch(Predicate<? super T> predicate)
import java.util.*;
import java.util.stream.*;

public class GFG {
    public static void main(String[] args) {
        List<String> list = Arrays.asList("Apple", "Banana", "Cherry");

        // Check if any string in the list starts with "B"
        boolean result = list.stream().anyMatch(s -> s.startsWith("B"));

        // Print the result
        System.out.println(result);  // Output: true
    }
}
true

Stream.allMatch()​

  • allMatch() is a terminal operation in Java Streams that checks whether all elements in the stream satisfy a given condition (predicate). It short-circuits, meaning it stops processing as soon as it finds an element that does not match the condition.
  • Purpose: It checks whether all elements in the stream satisfy the provided condition (predicate).
  • If any element doesn't match the condition, it returns false immediately.
  • If the stream is empty, it returns true (since an empty stream has no elements that violate the condition).
  • Predicate<? super T> predicate: The condition or test that each element in the stream must satisfy.
  • true if all elements match the predicate, or if the stream is empty.
  • false if any element does not match the predicate.
  • Like anyMatch(), allMatch() is short-circuiting, meaning it stops evaluating further once it finds an element that doesn't meet the condition.
  • If the stream is empty, it automatically returns true (since there are no elements to violate the condition).
  • allMatch() checks if all elements in the stream satisfy the condition. It returns true if all elements match, or if the stream is empty. If any element doesn't match, it immediately returns false.
boolean allMatch(Predicate<? super T> predicate)
import java.util.*;
import java.util.stream.*;

public class GFG {
    public static void main(String[] args) {
        List<Integer> list = Arrays.asList(3, 6, 9, 12);

        // Check if all numbers are divisible by 3
        boolean result = list.stream().allMatch(n -> n % 3 == 0);

        // Print the result
        System.out.println(result);  // Output: true
    }
}
// output
true

filter()​

  • The filter() method is used to select only the elements you want from a stream.
  • You give it a condition (called a predicate), and it keeps only the elements that match that condition.
  • filter() is an intermediate operation – it doesn’t run by itself.
  • It is lazy – it waits until you call a terminal method (like forEach, collect, etc.).
  • It returns a new stream that contains only the elements that passed the condition.

Syntax

Stream<T> filter(Predicate<? super T> predicate)
  • Stream<T> – The stream you’re working with (T means the type of data, like String, Integer, etc.)
  • Predicate – A function that returns true or false for each element (it's the condition).
  • Returns a new stream that contains only elements where the condition is true.

Example 1: filter() method with the operation of filtering out the elements divisible by 5.

public class StreamExample {
    public static void main(String[] args) {
        List<Integer> list = Arrays.asList(3, 4, 6, 12, 20);

        list.stream()
                .filter(num -> num%5 == 0)
                .forEach(System.out::println);
    }
}

// output
// 20

Example 2: filter() method with the operation of picking the elements with an upperCase letter at index 1.

public class StreamExample {
    public static void main(String[] args) {
        Stream<String> stream = Stream.of("Geeks", "fOr", "GEEKSQUIZ", "GeeksforGeeks");
        stream
                .filter(str -> Character.isUpperCase(str.charAt(1)))
                .forEach(System.out::println);
    }
}

// output

GEEKSQUIZ

Example 3: filter() method with the operation picking the elements ending with custom alphabetically letter say it be ‘s’ for implementation purposes.

public class StreamExample {

Stream.sorted(Comparator comparator)​

  • This method sorts the elements of a stream according to a provided Comparator
  • If the stream is ordered, the sort will maintain the relative order of elements with the same value (stable sort).
  • If the stream is unordered, no guarantee of stability in sorting.
  • It is a stateful operation, meaning it may need to remember elements to properly sort them.
Stream<T> sorted(Comparator<? super T> comparator)
  • Comparator<? super T> comparator: A comparison logic to determine how the elements should be sorted.
  • It returns a new stream where elements are sorted based on the given comparator.
import java.util.*;

public class GFG {
    public static void main(String[] args) {
        List<Integer> list = Arrays.asList(5, 2, 8, 1, 3);

        // Sort the list in ascending order
        list.stream().sorted((a, b) -> a - b).forEach(System.out::println);
    }
}

Stream.sorted()​

  • It sorts the elements of the stream according to their natural order (e.g., ascending order for numbers, alphabetical order for strings).
  • Stable for ordered streams: The order of elements that are equal will remain unchanged.
  • Unstable for unordered streams: The relative order of equal elements is not guaranteed.
  • It is a stateful operation, meaning it may require memory to track previously processed elements during sorting.
  • If the elements in the stream are not comparable, it will throw a java.lang.ClassCastException.
  • Return: A new stream where the elements are sorted in their natural order.
Stream<T> sorted()
import java.util.*;

        List<Integer> list = Arrays.asList(5, 2, 8, 1, 3);

        // Sort the list in natural order (ascending)
        list.stream().sorted().forEach(System.out::println);
    }
}

distinct()​

  • distinct() returns a stream with unique elements, removing duplicates.
  • It uses the hashCode() and equals() methods of the elements to check for duplicates.
  • For ordered streams, it keeps the first occurrence and maintains order (stable).
  • For unordered streams, the order of distinct elements is not guaranteed.
  • It’s a stateful intermediate operation, meaning it remembers seen elements while processing the stream.
Stream<T> distinct()
  • T is the type of stream elements.
  • Returns a stream with only distinct elements
import java.util.*;

public class GFG {

        List<Integer> list = Arrays.asList(2, 3, 2, 4, 3, 5);

        list.stream()
            .distinct()
            .forEach(System.out::println);
    }
}

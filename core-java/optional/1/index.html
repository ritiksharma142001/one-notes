Optional Class

GFG

What is Optional ?​

  • Introduced in Java 8, in java.util package.
  • A container object that may or may not contain a non-null value.
  • Helps avoid NullPointerException by providing a more expressive way to handle null.
  • Optional is like a box that may or may not contain a value.
  • It helps you avoid NullPointerException (when you try to use something that's null).
  • It is in the java.util package, so you need to import it.

How it works:

  • isPresent() → tells you if there's a value inside.
  • get() → gives you the value if it's there (but should be used carefully).
  • orElse(default) → gives the value if present, or a default value if not.
  • ifPresent(action) → runs some code only if the value is present.

Important About Optional:

  • It is a value-based class, meaning:
  • Once created, you cannot change it (immutable).
  • It compares using .equals(), not ==.
  • You can't create it using new, you must use methods like Optional.of() or Optional.empty().

Why Use Optional?

  • Prevents crashes due to null values.
  • Makes code more readable and expressive.
  • Avoids excessive null checks.
// Java program without Optional Class

public class OptionalDemo {
    public static void main(String[] args)
    {
        String[] words = new String[10];
        String word = words[5].toLowerCase();
        System.out.print(word);
    }
}

// output
// Exception in thread "main" java.lang.NullPointerException

To avoid abnormal termination, we use the Optional class. In the following example, we are using Optional. So, our program can execute without crashing

Static Methods in Java (Basics)

  • What is a Static Method?
  • A static method belongs to the class, not to objects.
  • You can call it without making an object of the class.

Syntax:

public static void greet(String name) {
   System.out.println("Hello " + name);
}

Key Points:

  • Use the static keyword in the method declaration.
  • You can call it using the class name, like MyClass.greet("John");.
  • Can be overloaded (same name, different parameters).
  • Cannot be overridden (no runtime polymorphism).
  • Shared among all instances of a class.

Simple Real-life Analogy:

  • Optional is like a box that may or may not have a gift inside. You check before opening.
  • Static methods are like tools in a toolbox — you don’t need to "build" a tool (object), you just use it directly.

Optional.empty()​

Use when you want to create an empty Optional (no value inside)

Optional<String> emptyOpt = Optional.empty();
System.out.println(emptyOpt.isPresent());  // false

Optional.of(value)​

Use when you are sure the value is not null.

Optional<String> nameOpt = Optional.of("Alice");
System.out.println(nameOpt.get());  // Alice

If you pass null, it will throw NullPointerException

Optional<String> error = Optional.of(null);  // ❌ Throws NPE

Optional.ofNullable(value)​

Use when the value might be null.

String name = null;
Optional<String> maybeName = Optional.ofNullable(name);

System.out.println(maybeName.isPresent());  // false

If name had a value:

String name = "Bob";
Optional<String> maybeName = Optional.ofNullable(name);
System.out.println(maybeName.get());  // Bob

What are Instance Methods?​

  • Instance methods are regular methods that belong to an object, not the class.
  • To use them, you must create an object of the class first.
public class MyClass {
    public void greet(String name) {
        System.out.println("Hello " + name);
    }
}

MyClass obj = new MyClass();  // create object
obj.greet("Alice");           // call instance method

Key Points:​

FeatureDescription
Belongs to objectYou need to create an instance (object) to use it.
Own copy for each objectEach object can access the method with its own data.
Can be overriddenSubclasses can provide their own version.
Uses dynamic bindingThe method to run is decided at runtime, not compile time.

  1. equals(Object o) Compares an Optional object to another object to see if they are equal.

Returns true if both have the same value (if present) and are of the same type.

Optional<String> opt1 = Optional.of("Hello");
Optional<String> opt2 = Optional.of("Hello");
System.out.println(opt1.equals(opt2));  // true
  1. filter(Predicate<? super T> predicate) Applies the given predicate (condition) to the value if present.

Returns the Optional itself if the value matches the predicate, otherwise returns Optional.empty().

Optional<String> opt = Optional.of("Hello");
Optional<String> filtered = opt.filter(s -> s.length() > 3);
System.out.println(filtered.get());  // Hello
  1. flatMap(Function<? super T, Optional<U>> mapper) Similar to map(), but returns an Optional directly (i.e., you avoid nesting Optional inside another Optional).
Optional<String> opt = Optional.of("Hello");
Optional<Integer> result = opt.flatMap(s -> Optional.of(s.length()));
System.out.println(result.get());  // 5
  1. get() Retrieves the value contained in the Optional (if present).

Throws NoSuchElementException if the value is not present.

Optional<String> opt = Optional.of("Hello");
System.out.println(opt.get());  // Hello

⚠️ Be careful! If Optional is empty, get() throws an exception.

  1. hashCode() Returns the hash code of the value contained in the Optional.

If empty, returns 0.

Optional<String> opt = Optional.of("Hello");
System.out.println(opt.hashCode());  // Hash code based on "Hello"
  1. ifPresent(Consumer<? super T> action) Executes the given action if the value is present.

If the value is not present, does nothing.

Optional<String> opt = Optional.of("Hello");
opt.ifPresent(s -> System.out.println("Value: " + s));  // Value: Hello
  1. isPresent() Returns true if the Optional contains a value, false if empty.
Optional<String> opt = Optional.of("Hello");
System.out.println(opt.isPresent());  // true
  1. map(Function<? super T, ? extends U> mapper) Transforms the value if present using the given function.

If the value is absent, it returns an empty Optional.

Optional<String> opt = Optional.of("Hello");
Optional<String> mapped = opt.map(s -> s.toUpperCase());
System.out.println(mapped.get());  // HELLO
  1. orElse(T other) Returns the contained value if present, otherwise returns the given default value.
Optional<String> opt = Optional.empty();
System.out.println(opt.orElse("Default Value"));  // Default Value
  1. orElseGet(Supplier<? extends T> other) Similar to orElse(), but instead of passing a value, you provide a function that generates the value.
Optional<String> opt = Optional.empty();
System.out.println(opt.orElseGet(() -> "Generated Default Value"));  // Generated Default Value
  1. orElseThrow(Supplier<? extends X> exceptionSupplier) Returns the value if present, or throws the exception generated by the provided exception supplier if absent.
Optional<String> opt = Optional.empty();
System.out.println(opt.orElseThrow(() -> new IllegalArgumentException("Value not present")));  // Throws exception
  1. toString() Returns a string representation of the Optional object.

If present, it shows the value; if not, it shows Optional.empty.

Optional<String> opt = Optional.of("Hello");
System.out.println(opt.toString());  // Optional[Hello]

Optional<String> emptyOpt = Optional.empty();
System.out.println(emptyOpt.toString());  // Optional.empty

Concrete Methods:​

A concrete method means, the method has a complete definition but can be overridden in the inherited class. If we make this method final, then it can not be overridden. Declaring method or class “final” means its implementation is complete. It is compulsory to override abstract methods. Concrete Methods can be overridden in the inherited classes if they are not final. The following table shows the list of Concrete Methods provided by the Optional Class :

✅ Why Concrete Methods Matter in Optional?​

Flexibility: You can use the provided concrete methods directly without needing to implement them yourself.

Override Possibility: If the method isn't final, you can override it in subclasses to provide custom behavior.

final Keyword: If a method is declared as final, you cannot override it in subclasses (this is often done to lock down the behavior for safety or performance reasons).

  1. empty() Description: Returns an empty Optional instance (i.e., no value inside).

Usage: Useful for representing a value that is absent.

Optional<String> emptyOpt = Optional.empty();
System.out.println(emptyOpt.isPresent());  // false
  1. equals(Object o) Description: Compares the current Optional instance to another object.

Usage: Checks if two Optional instances are equal, including the value inside.

Optional<String> opt1 = Optional.of("Hello");
Optional<String> opt2 = Optional.of("Hello");
System.out.println(opt1.equals(opt2));  // true
  1. filter(Predicate<? super T> predicate) Description: Filters the value inside the Optional using the provided condition (Predicate).

Usage: Returns an Optional containing the value if it matches the condition, or an empty Optional if it doesn't.

Optional<String> opt = Optional.of("Hello");
Optional<String> filtered = opt.filter(s -> s.length() > 3);
System.out.println(filtered.get());  // Hello
  1. flatMap(Function<? super T, Optional<U>> mapper) Description: Similar to map(), but the function returns an Optional itself.

Usage: Avoids nested Optional objects by directly transforming the value inside.

Optional<String> opt = Optional.of("Hello");
Optional<Integer> length = opt.flatMap(s -> Optional.of(s.length()));
System.out.println(length.get());  // 5
  1. get() Description: Retrieves the value inside the Optional if present.

Usage: Throws NoSuchElementException if the value is not present.

Optional<String> opt = Optional.of("Hello");
System.out.println(opt.get());  // Hello

⚠️ Warning: If the Optional is empty, get() throws an exception.

  1. hashCode() Description: Returns the hash code of the value inside the Optional, or 0 if it's empty.

Usage: Used for comparing Optional objects, for example, in collections.

Optional<String> opt = Optional.of("Hello");
System.out.println(opt.hashCode());  // Hash code of "Hello"
  1. ifPresent(Consumer<? super T> action) Description: Executes the provided action if the value is present.

Usage: Runs a block of code only if a value is present inside the Optional.

Optional<String> opt = Optional.of("Hello");
opt.ifPresent(s -> System.out.println("Value: " + s));  // Value: Hello
  1. isPresent() Description: Returns true if the value is present, otherwise false.

Usage: Checks if the Optional contains a value or is empty.

Optional<String> opt = Optional.of("Hello");
System.out.println(opt.isPresent());  // true
  1. map(Function<? super T, ? extends U> mapper) Description: Transforms the value inside the Optional using the provided function.

Usage: If the value is present, applies the function to it; otherwise, returns an empty Optional.

Optional<String> opt = Optional.of("Hello");
Optional<String> upper = opt.map(s -> s.toUpperCase());
System.out.println(upper.get());  // HELLO
  1. of(T value) Description: Creates an Optional with the provided non-null value.

Usage: Used when you are certain the value is not null.

Optional<String> opt = Optional.of("Hello");
System.out.println(opt.get());  // Hello

⚠️ Warning: If the value is null, Optional.of() throws a NullPointerException.

  1. ofNullable(T value) Description: Creates an Optional that may or may not contain the provided value (null-safe).

Usage: If the value is null, it returns an empty Optional; otherwise, it contains the value.

Optional<String> opt = Optional.ofNullable(null);
System.out.println(opt.isPresent());  // false
  1. orElse(T other) Description: Returns the value if present, otherwise returns the provided default value.

Usage: Useful when you need a fallback value.

Optional<String> opt = Optional.empty();
System.out.println(opt.orElse("Default Value"));  // Default Value
  1. orElseGet(Supplier<? extends T> other) Description: Similar to orElse(), but uses a supplier (a function that generates the default value) to return a value.

Usage: Useful when generating the default value is costly or requires computation.

Optional<String> opt = Optional.empty();
System.out.println(opt.orElseGet(() -> "Generated Default"));  // Generated Default
  1. orElseThrow(Supplier<? extends X> exceptionSupplier) Description: Returns the value if present, or throws the exception created by the provided supplier if the value is absent.

Usage: Useful for throwing a custom exception when the value is not present.

Optional<String> opt = Optional.empty();
try {
    opt.orElseThrow(() -> new IllegalArgumentException("Value is missing"));
} catch (Exception e) {
    System.out.println(e.getMessage());  // Value is missing
}
  1. toString() Description: Returns a string representation of the Optional.

Usage: If the value is present, it shows the value; if empty, it shows Optional.empty.

Optional<String> opt = Optional.of("Hello");
System.out.println(opt.toString());  // Optional[Hello]

Optional<String> emptyOpt = Optional.empty();
System.out.println(emptyOpt.toString());  // Optional.empty

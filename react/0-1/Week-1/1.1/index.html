Introduction to Javascript, data types, varibles and if-else conditions

  • Notes

JSON​

  • JSON stands for JavaScript Object Notation. It is used to store and exchange data.

Accessing Data from JSON​

  • If you have a JavaScript object, you can access data using:
const user = {
    name: 'Alice',
    age: 25,
}

console.log(user.name) // Alice
console.log(user['age']) // 25

JSON.parse() Method​

  • JSON.parse() is used to convert a JSON string into a JavaScript object.
const jsonData = '{"name":"Bob","age":30}'

const obj = JSON.parse(jsonData) // converts to JavaScript object
console.log(obj) // {name: 'Bob', age: 30}
console.log(obj.name) // Bob
console.log(obj.age) // 30

JSON.stringify() Method​

  • JSON.stringify() is used to convert a JavaScript object into a JSON string.

const obj = {
  name: "Charlie",
  age: 28
};

const jsonString = JSON.stringify(obj);

console.log(jsonString); // {"name":"Charlie","age":28}

fetch​

  • fetch data from server.

Get Request​

async function getPost() {
  try {
    const response = await fetch('https://jsonplaceholder.typicode.com/posts/1');
    const data = await response.json();
    console.log(data);
  } catch (error) {
    console.error('Error:', error);
  }
}

getPost();

// {userId: 1, id: 1, title: 'sunt aut facere repellat provident occaecati excepturi optio reprehenderit', body: 'quia et suscipit\nsuscipit recusandae consequuntur …strum rerum est autem sunt rem eveniet architecto'}

post​

  • send data to the server.
fetch('https://jsonplaceholder.typicode.com/posts', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    title: 'Hello World',
    body: 'This is a test post',
    userId: 1
  })
})
.then(response => response.json())
.then(data => console.log('POST Success:', data))
.catch(error => console.error('Error:', error));

// POST Success: {title: 'Hello World', body: 'This is a test post', userId: 1, id: 101}

put​

  • Update Data Fully
fetch('https://jsonplaceholder.typicode.com/posts/1', {
  method: 'PUT',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    id: 1,
    title: 'Updated Title',
    body: 'Updated body content',
    userId: 1
  })
})
.then(response => response.json())
.then(data => console.log('PUT Success:', data))
.catch(error => console.error('Error:', error));

// PUT Success: {id: 1, title: 'Updated Title', body: 'Updated body content', userId: 1}

patch​

  • Update Data Partially.
fetch('https://jsonplaceholder.typicode.com/posts/1', {
  method: 'PATCH',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    title: 'Only Title Updated'
  })
})
.then(response => response.json())
.then(data => console.log('PATCH Success:', data))
.catch(error => console.error('Error:', error));

// PATCH Success: {userId: 1, id: 1, title: 'Only Title Updated', body: 'quia et suscipit\nsuscipit recusandae consequuntur …strum rerum est autem sunt rem eveniet architecto'}

delete​

  • Remove Data
fetch('https://jsonplaceholder.typicode.com/posts/1', {
  method: 'DELETE'
})
.then(response => {
  if (response.ok) {
    console.log('DELETE Success');
  } else {
    console.error('Delete Failed');
  }
})
.catch(error => console.error('Error:', error));

// DELETE Success

What is a Query String?​

  • A query string is the part of a URL that passes data to the server. It starts with ? and has key-value pairs separated by &.

How to Create a Query String ?​

Manually const url = "https://example.com/search?name=John&age=25";

Using URLSearchParams

const params = new URLSearchParams({
    name: 'John',
    age: 25,
    city: 'Delhi',
})
const url = `https://example.com/profile?${params.toString()}`
console.log(url) // https://example.com/profile?name=John&age=25&city=Delhi

How to Read Query Strings in JavaScript ?​

// const url = new URL(window.location.href)
const url = new URL('https://example.com/profile?name=John&age=25&city=Delhi')

const params = new URLSearchParams(url.search)
console.log(params)

console.log(params.get("name"));  // Output: "John"
console.log(params.get("age"));   // Output: "25"

What is the use of Content-Type header?

  • The Content-Type header tells the server what kind of data you are sending in the body of the HTTP request.
headers: {
  'Content-Type': 'application/json'
}
  • The request body contains JSON data.
  • So the server knows to parse the body as JSON.

Why is this important?

  • If the server expects JSON but you don’t set Content-Type: application/json, it might fail to parse the request body correctly.
  • For example, if you send data as JSON without this header, the server may think it’s plain text or some other format and reject or misinterpret it.

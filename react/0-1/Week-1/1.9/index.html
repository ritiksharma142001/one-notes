Fetch, API, Promises and Async-Await in JS

  • Third Party Dummy Json API
  • Notes

Fetch​

  • fetch is not a web browser feature (a Web API).
  • Browsers like Chrome, Firefox, etc. give JavaScript the ability to use fetch.
  • Node.js older version dont have fetch but latest version have it like v18+.

How fetch works ?​

  • When we fetch something from the server, we do not get the result immediately.
  • The fetch function returns a Promise because there will be some latency.
  • A Promise means: “When I get the result back from the server, I’ll give it to you.”
  • Here, the Promise is like a person saying this. So, the Promise is the one who gives the result back to you when it's ready.

Why do we use Web APIs to make third-party API calls?​

  • We use Web APIs like fetch() to call third-party services so that:

  1. The client doesn't hang or lag.

    • What happens if we block the main thread?
    • The page may freeze and become unresponsive.

  2. The client (browser) stays free to do other tasks.

    • What can the browser do while waiting for data?
    • It can show a loader or let the user interact with the page.

  3. Blocking operations are handled in the background.

    • Who performs these background tasks?
    • The browser handles them using fetch() and Web APIs.

  4. We don’t block the main thread.

    • Why is it important not to block the main thread?
    • Because it controls user interface updates and interactions.

  5. As soon as we get the result (Promise is fulfilled), we show it to the user.

    • What is a Promise in JavaScript?
    • A Promise is a way to handle operations that take time, like fetching data from a server.

Basic fetch code​

code-1 API Fetching​

console.log("Calling the API")

const response = fetch("https://dummyjson.com/users");
response.then((data)=>{
    console.log("Data is ", data)
}).catch((err)=>{
    console.log("Error is ", err)
})

console.log("Got the response ", response);

code 3 - Display Users id and firstname​

console.log("Calling the API")

const response = fetch("https://dummyjson.com/users");
response.then((data)=>{
    console.log("Data is ", data)
    const users = data.json()
    console.log(users)
    users.then((res)=>{
        console.log("Got the users", res)
        const users = res.users
        for(let user of users){
            console.log(user.id, user.firstName)
        }
    }).catch((err)=>{
        console.log("error is ", err)
    })
}).catch((err)=>{
    console.log("Error is ", err)
})

console.log("Got the response ", response);

Promise creation​

  • To return a Promise, fetch must created a promise internally.
  • Syntax
  • In resolve, we can just send one data.
  • either string, json.
  • we can combine multiple data as object and send.
let promise = new Promise(function(resolve, reject){
    //do something
});

code-1 Basic​

const promise = new Promise((resolve, reject)=>{
    setTimeout(()=>{
        console.log("Hello")
        resolve("secret data")
    }, 5000)
})
promise.then((data)=>{
    console.log("Got the data",data)
})

code-2 resolve-reject​

const promise = new Promise((resolve, reject) => {
  const allGood = false;
  setTimeout(() => {
    console.log("Hello");
    if (allGood) {
      resolve("secret data");
    } else {
      reject("All was not Good");
    }
  }, 5000);
});
promise
  .then((data) => {
    console.log("Got the data", data);
  })
  .catch((err) => {
    console.log("Got the error", err);
  });

code-3 Promise chaining​

const res = fetch("https://dummyjson.com/users")
  .then((data) => {
    console.log("got the data", data);
    return data.json();
  })
  .then((data) => {
    console.log("Got the json data", data);
  })
  .catch((err) => {
    console.log("got the error", err);
  })
  .finally(() => {
    console.log("finally always be executed");
  });

            console.log("Booking initiated")
            resolve({
                bookingId: "76jdfvbhsd",

            resolve(booking)
        }, 3000)
    })
}

    return new Promise((resolve, reject)=>{
        setTimeout(()=>{
            console.log("Payment processed")
            booking["payment"] = payment
            resolve(booking)
        }, 3000)
    })
}
